Uber / Ride-Sharing Platform Design

Uber is a real-time geospatial matching system. The core challenges are matching riders with nearby drivers in seconds, handling 250K location updates per second, and implementing dynamic surge pricing based on supply and demand.

Define: Requirements

  • Functional: Request ride (pickup + destination), match with nearest available driver, real-time tracking (rider sees driver on map), dynamic pricing (surge), automatic payment, rating system, ETA estimation
  • Non-Functional: Match within 30 seconds (p95), driver location updates every 3-4 seconds, tracking latency <1 second, 99.99% availability, support 5M concurrent drivers
  • Out of Scope: Uber Eats, scheduled rides, carpooling

Estimate

# 100M MAU, 20M rides/day, 5M drivers (1M active at peak)

# RIDE REQUESTS:
#   20M / 86400 = 230 QPS (peak ~700)
#   Not that high! Challenge is MATCHING, not throughput.

# DRIVER LOCATION UPDATES:
#   1M drivers x 1 update/4 sec = 250K updates/sec
#   THIS is the high-volume operation
#   Each update: {driver_id, lat, lng, timestamp} ~ 50 bytes

# TRACKING:
#   500K concurrent rides
#   500K x 0.5 updates/sec = 250K WebSocket pushes/sec

# STORAGE:
#   Ride history: 20M/day x 365 x 2 KB = 14.6 TB/year
#   Location traces: 250K/sec x 50B x 86400 = 1 TB/day
#   Keep 30 days hot, archive to cold storage

Geospatial Indexing with Geohash

import math

def geohash_encode(lat: float, lng: float, precision: int = 6) -> str:
    """Encode lat/lng to geohash string."""
    BASE32 = "0123456789bcdefghjkmnpqrstuvwxyz"
    lat_range, lng_range = [-90.0, 90.0], [-180.0, 180.0]
    bits, is_lng = [], True
    while len(bits) < precision * 5:
        if is_lng:
            mid = (lng_range[0] + lng_range[1]) / 2
            if lng >= mid:
                bits.append(1); lng_range[0] = mid
            else:
                bits.append(0); lng_range[1] = mid
        else:
            mid = (lat_range[0] + lat_range[1]) / 2
            if lat >= mid:
                bits.append(1); lat_range[0] = mid
            else:
                bits.append(0); lat_range[1] = mid
        is_lng = not is_lng
    # Convert bits to BASE32 characters
    result = []
    for i in range(0, len(bits), 5):
        chunk = bits[i:i+5]
        idx = sum(b << (4-j) for j, b in enumerate(chunk))
        result.append(BASE32[idx])
    return "".join(result)

Matching Algorithm

  • Geohash index: Drivers indexed by geohash prefix. Finding nearby drivers = query same geohash + neighbors.
  • Candidate selection: Find all available drivers in nearby geohash cells
  • Ranking: Score candidates by distance, ETA, driver rating, acceptance rate
  • Offer cascade: Send ride offer to top driver first. If declined/timeout, try next driver.

Surge Pricing

  • Divide city into hexagonal zones (H3 cells)
  • Calculate supply (available drivers) and demand (ride requests) per zone
  • Surge multiplier = demand / supply, capped at a maximum (e.g., 3x)
  • Update every 2-5 minutes to respond to changing conditions

Architecture

  • Location Service: Ingests 250K updates/sec, updates geohash index in Redis
  • Matching Service: Queries geohash index, runs ranking algorithm, sends offers
  • Trip Service: Manages active trips, calculates fares, handles payments
  • ETA Service: Graph-based routing with real-time traffic data
  • Pricing Service: Dynamic surge calculation per zone

Key Design Decisions

  • Geohash precision 6: ~1 km cells balance precision with query efficiency
  • Location data is ephemeral: Only current positions matter for matching; historical data archived
  • WebSocket for tracking: Push-based updates for real-time rider experience
  • Eventual consistency for location: A few seconds of stale location data is acceptable for matching
