Search Engines and Elasticsearch

Why SQL LIKE '%query%' doesn't scale, and how inverted indexes work. The core of Google, Elasticsearch, and every search bar you've ever used.

The Problem: LIKE '%query%' Fails

  • Full table scan: reads every row, no index can help with leading wildcards
  • "wireless mouse" won't match "mouse wireless" (word order)
  • "running shoes" won't match "runners shoe" (no stemming)
  • No relevance ranking, no typo tolerance, no synonyms

Inverted Index: The Core Data Structure

A forward index maps documents to words. An inverted index maps words to documents.

from collections import defaultdict
from typing import Dict, List, Set
import re
import math

class InvertedIndex:
    """Simplified search engine with inverted index."""

    def __init__(self):
        self.index: Dict[str, Set[int]] = defaultdict(set)
        self.documents: Dict[int, str] = {}
        self.doc_count = 0

    def _tokenize(self, text: str) -> List[str]:
        """Split text into searchable terms."""
        text = text.lower()
        text = re.sub(r'[^\w\s]', '', text)  # Remove punctuation
        tokens = text.split()
        # Remove stop words
        stop_words = {"the", "a", "an", "is", "and", "or", "in", "to"}
        return [t for t in tokens if t not in stop_words]

    def add_document(self, doc_id: int, text: str):
        """Index a document."""
        self.documents[doc_id] = text
        self.doc_count += 1
        for term in self._tokenize(text):
            self.index[term].add(doc_id)

    def search(self, query: str) -> List[int]:
        """Search: find documents containing ALL query terms."""
        terms = self._tokenize(query)
        if not terms:
            return []
        result = self.index.get(terms[0], set()).copy()
        for term in terms[1:]:
            result &= self.index.get(term, set())
        return sorted(result)

Building an Inverted Index

  • Tokenize: Split text into terms ("Hello World!" -> ["hello", "world"])
  • Normalize: Lowercase, remove punctuation
  • Stem: Reduce to root form ("running" -> "run", "mice" -> "mouse")
  • Remove stop words: Drop "the", "a", "is", "and" (optional)
  • Build: For each term, record which documents contain it

TF-IDF Relevance Scoring

Not all matches are equal. TF-IDF ranks results by relevance:

  • Term Frequency (TF): How often the term appears in this document
  • Inverse Document Frequency (IDF): How rare the term is across all documents. Rare terms are more important
  • Score = TF * IDF. Documents with rare, frequently-occurring query terms rank higher

Elasticsearch Architecture

  • Index: Like a database table. Contains documents of a similar type
  • Shards: Each index is split into shards for distributed storage and parallel queries
  • Replicas: Each shard has replicas for fault tolerance and read scaling
  • Analyzers: Configurable text processing pipelines (tokenization, stemming, synonyms)

When to Use a Search Engine

  • Full-text search with relevance ranking
  • Autocomplete and suggestions
  • Faceted search (filter by category, price range, brand)
  • Log analysis and aggregation (ELK stack: Elasticsearch + Logstash + Kibana)
  • Do NOT use as a primary database -- use it as a secondary index synced from your source of truth
