Project: Web Crawler Prototype

This project implements a mini web crawler with BFS crawling, link extraction, URL deduplication via Bloom filter simulation, content deduplication via SimHash, per-domain politeness rate limiting, and a search index for crawled pages.

URL Deduplication with Bloom Filter

class BloomFilter:
    """Probabilistic set membership with no false negatives."""
    def __init__(self, expected_items: int, fp_rate: float = 0.01):
        import math
        self.size = int(-expected_items * math.log(fp_rate)
                        / (math.log(2) ** 2))
        self.num_hashes = int((self.size / expected_items)
                              * math.log(2))
        self.bits = [False] * self.size

    def add(self, item: str):
        for seed in range(self.num_hashes):
            idx = hash(f"{seed}:{item}") % self.size
            self.bits[idx] = True

    def might_contain(self, item: str) -> bool:
        return all(
            self.bits[hash(f"{seed}:{item}") % self.size]
            for seed in range(self.num_hashes)
        )

Politeness: Per-Domain Rate Limiting

  • Each domain has a separate crawl queue
  • Maximum 1 request per second per domain enforced
  • Crawler round-robins across domain queues to maximize throughput while respecting limits
  • Robots.txt rules checked before fetching any page

Crawl Process (BFS)

  • Initialize: Seed URLs added to frontier with high priority
  • Fetch: Pick next URL from frontier (respecting priority and politeness)
  • Parse: Extract page title, content, and outgoing links
  • Dedup: Check Bloom filter for each extracted link. Skip already-seen URLs.
  • Content dedup: Compute SimHash fingerprint. Skip near-duplicate content.
  • Index: Store page content and metadata in search index
  • Enqueue: Add new (unseen) links to frontier

Search Index

After crawling, pages are indexed for simple keyword search. The index maps words to lists of pages containing them, along with relevance scores based on term frequency and page importance.

Key Takeaways

  • Bloom filters enable efficient URL deduplication with controllable false positive rates
  • BFS crawl order naturally discovers important (well-linked) pages first
  • Per-domain rate limiting is essential for being a good web citizen
  • Content deduplication via fingerprinting catches mirror sites and syndicated content
  • The prototype demonstrates all core crawler components that scale to billions of pages with distributed infrastructure
