Exercise 2: Message Queue Selection

Choose between Kafka, RabbitMQ, and SQS for each scenario. Explain your reasoning.

  • Scenario A: You need to distribute image resize jobs across 50 worker processes. Jobs should be picked up by exactly one worker. You need priority queues (premium users get processed first)
  • Scenario B: You're building a real-time analytics pipeline. Click events from 100,000 concurrent users need to be processed by three independent consumer groups: real-time dashboard, fraud detection, and data warehouse loader
  • Scenario C: You're running on AWS and need a simple queue to decouple your API from a background job processor. You want zero operational overhead
# Decision matrix:
# Feature           | RabbitMQ     | Kafka         | SQS
# ─────────────────┼──────────────┼───────────────┼───────────
# Routing           | Complex      | Partition key | Simple
# Consumer groups   | No           | Yes           | No
# Message replay    | No           | Yes           | No
# Priority queues   | Yes          | No            | No (sort of)
# Ops overhead      | Medium       | High          | Zero
# Throughput        | ~50K msg/s   | ~1M msg/s     | ~3K FIFO

Exercise 3: Making Operations Idempotent

The following operations are NOT idempotent. Redesign each one so that processing the same message twice produces the same result.

  • balance += 100 (crediting a user's account)
  • INSERT INTO orders (user_id, amount, created_at) VALUES (...)
  • Sending a welcome email to a new user
# Example fix for balance credit:
def credit_account_idempotent(user_id, amount, transaction_id, db):
    """Idempotent credit using transaction_id as dedup key."""
    # Check if already processed
    existing = db.query(
        "SELECT 1 FROM transactions WHERE id = %s", transaction_id
    )
    if existing:
        return  # Already processed, skip

    # Process within a transaction
    db.execute("BEGIN")
    db.execute(
        "INSERT INTO transactions (id, user_id, amount) VALUES (%s, %s, %s)",
        transaction_id, user_id, amount
    )
    db.execute(
        "UPDATE accounts SET balance = balance + %s WHERE user_id = %s",
        amount, user_id
    )
    db.execute("COMMIT")

# Now design similar fixes for the other two operations.
