Exercise 1: Monolith vs Microservices Decision

Your company has a monolith serving 10,000 requests/second. Evaluate whether to migrate to microservices for each scenario:

  • Scenario A: 3-person startup. One codebase. All features ship together weekly. Traffic is growing 20% per month
  • Scenario B: 80-person engineering team. Teams step on each other's code daily. The payment module needs 10x more resources than the user module. Deploys take 2 hours due to testing everything
  • Scenario C: 15-person team. The ML recommendation engine needs Python/TensorFlow. The rest is Java. The ML model retrains daily and needs GPU instances
# Decision framework:
def should_use_microservices(
    team_size: int,
    independent_scaling_needed: bool,
    independent_deploy_needed: bool,
    different_tech_stacks: bool,
    operational_maturity: str,  # "low", "medium", "high"
) -> str:
    if team_size < 10 and not independent_scaling_needed:
        return "MONOLITH -- too small, overhead not justified"
    if operational_maturity == "low":
        return "MODULAR MONOLITH -- get the boundaries right first"
    if independent_scaling_needed or team_size > 30:
        return "MICROSERVICES -- clear benefits outweigh complexity"
    return "MODULAR MONOLITH -- wait for a stronger signal"

Exercise 2: Design a Saga

  • Define the saga steps (forward actions)
  • Define the compensating transaction for each step
  • What happens if the hotel booking succeeds but the car rental fails?
  • Should you use choreography or orchestration? Why?
  • How do you handle the case where a compensating transaction itself fails?

]
