Design: Collaborative Editor

Design a real-time collaborative document editor like Google Docs.

Requirements

  • Multiple users edit the same document simultaneously
  • Real-time cursor and selection sync across clients
  • Conflict resolution when concurrent edits target the same region
  • Offline editing with eventual consistency upon reconnect

High-Level Architecture

  • Client Editor — Captures local operations and applies remote ones
  • WebSocket Gateway — Persistent connection per user for low-latency sync
  • Collaboration Server — Applies OT or CRDT merge logic, broadcasts ops
  • Document Store — Persistent storage of latest document state and op log

Key Components

# Operational Transformation (OT) - simplified
class OTServer:
    def __init__(self):
        self.doc = ""
        self.revision = 0
        self.op_log = []

    def apply_op(self, op, client_rev):
        # Transform op against all ops since client_rev
        for past_op in self.op_log[client_rev:]:
            op = self._transform(op, past_op)
        self.doc = self._execute(self.doc, op)
        self.op_log.append(op)
        self.revision += 1
        return op  # broadcast transformed op to others

    def _transform(self, op_a, op_b):
        """Adjust op_a's position given op_b happened first."""
        if op_a["type"] == "insert" and op_b["type"] == "insert":
            if op_a["pos"] > op_b["pos"]:
                op_a["pos"] += len(op_b["text"])
        return op_a

    def _execute(self, doc, op):
        if op["type"] == "insert":
            return doc[:op["pos"]] + op["text"] + doc[op["pos"]:]
        if op["type"] == "delete":
            return doc[:op["pos"]] + doc[op["pos"] + op["len"]:]
        return doc

Scaling Considerations

  • OT requires a central server per document; CRDT allows peer-to-peer and offline
  • Partition by document ID — each doc lives on one collaboration server
  • Cursor positions sent as presence updates on a separate lightweight channel
  • Periodic snapshots reduce op log replay time on reconnect
