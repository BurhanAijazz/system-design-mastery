Design: Distributed Web Crawler

Design a web crawler that can index billions of pages across the internet.

Requirements

  • Crawl billions of pages with BFS or priority-based ordering
  • Respect robots.txt and politeness (delay between requests to same host)
  • Deduplicate URLs and content
  • Horizontally scalable with fault tolerance

High-Level Architecture

  • URL Frontier — Priority queue of URLs to crawl, partitioned by domain
  • Fetcher Workers — Distributed pool that downloads pages via HTTP
  • DNS Resolver Cache — Local cache to avoid repeated DNS lookups
  • Content Parser — Extracts links, text, and metadata from HTML
  • Dedup Service — Bloom filter for URL dedup, simhash for content dedup
  • Storage — Raw HTML in blob store, parsed data in search index

Key Components

from collections import deque
import hashlib, mmh3, bitarray

class BloomFilter:
    def __init__(self, size=10_000_000, num_hashes=7):
        self.bits = bitarray.bitarray(size)
        self.bits.setall(0)
        self.size = size
        self.num_hashes = num_hashes

    def add(self, url):
        for seed in range(self.num_hashes):
            idx = mmh3.hash(url, seed) % self.size
            self.bits[idx] = 1

    def might_contain(self, url):
        return all(
            self.bits[mmh3.hash(url, seed) % self.size]
            for seed in range(self.num_hashes)
        )

class URLFrontier:
    def __init__(self):
        self.queues = {}  # domain -> deque of URLs
        self.bloom = BloomFilter()

    def add_url(self, url, domain):
        if not self.bloom.might_contain(url):
            self.bloom.add(url)
            self.queues.setdefault(domain, deque()).append(url)

Scaling Considerations

  • Partition frontier by domain hash so each worker owns a set of domains
  • Per-domain rate limiting ensures politeness (1-2 sec delay between requests)
  • Checkpoint frontier state to survive worker crashes
  • Use consistent hashing to rebalance domains when workers join/leave
