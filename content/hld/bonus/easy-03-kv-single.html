Design: Single-Node Key-Value Store

Design an in-memory key-value store on a single node with persistence and TTL support.

Requirements

  • Get/Put/Delete operations with O(1) average time
  • Durability via write-ahead log (WAL) and periodic snapshots
  • TTL-based key expiry
  • Recovery from crash using WAL replay on top of last snapshot

High-Level Architecture

  • In-Memory Engine — HashMap for data, separate HashMap for TTL metadata
  • WAL — Append-only file; every mutation logged before acknowledgement
  • Snapshot — Periodic full dump of HashMap to disk; truncate WAL after snapshot
  • Expiry Daemon — Background thread that lazily or actively evicts expired keys

Key Components

import time, threading

class KVStore:
    def __init__(self):
        self.data = {}
        self.ttls = {}  # key -> expiry_timestamp
        self.wal = open("wal.log", "a")

    def put(self, key, value, ttl=None):
        self._write_wal("PUT", key, value, ttl)
        self.data[key] = value
        if ttl:
            self.ttls[key] = time.time() + ttl

    def get(self, key):
        if key in self.ttls and time.time() > self.ttls[key]:
            self.delete(key)
            return None
        return self.data.get(key)

    def delete(self, key):
        self._write_wal("DEL", key)
        self.data.pop(key, None)
        self.ttls.pop(key, None)

    def _write_wal(self, op, key, value=None, ttl=None):
        self.wal.write(f"{op}|{key}|{value}|{ttl}\n")
        self.wal.flush()

    def snapshot(self):
        with open("snapshot.dat", "w") as f:
            for k, v in self.data.items():
                f.write(f"{k}|{v}|{self.ttls.get(k,'')}\n")
        open("wal.log", "w").close()  # truncate WAL

Scaling Considerations

  • Single-node bottleneck — shard across nodes for horizontal scale
  • WAL fsync trade-off: fsync every write (durable) vs. batched (faster)
  • Lazy expiry on read + periodic active sweep balances CPU and memory
  • Snapshot uses fork() / copy-on-write to avoid blocking writes (Redis approach)
