Design: S3 Object Storage

Design a scalable object storage service similar to Amazon S3.

Requirements

  • PUT/GET/DELETE objects up to 5TB in size
  • Multipart upload for large objects
  • 11 nines (99.999999999%) durability via redundancy
  • Object versioning and lifecycle policies
  • Bucket-level access control and pre-signed URLs

High-Level Architecture

  • API Gateway — REST API: PUT /bucket/key, GET /bucket/key
  • Metadata Service — Stores object metadata (key, size, version, location pointers)
  • Placement Service — Decides which data nodes store each object's chunks
  • Data Nodes — Store raw data blocks on local disks
  • Erasure Coding — Split data into k data + m parity shards (e.g., 8+4)

Key Components

class MetadataService:
    def __init__(self, db):
        self.db = db

    def put_object(self, bucket, key, size, locations, version=None):
        obj = {"bucket": bucket, "key": key, "size": size,
               "locations": locations,
               "version": version or self._next_version(bucket, key)}
        self.db.upsert(f"{bucket}/{key}", obj)

    def get_object(self, bucket, key, version=None):
        return self.db.get(f"{bucket}/{key}", version=version)

class ErasureCoder:
    def encode(self, data, k=8, m=4):
        """Split data into k data shards + m parity shards."""
        shard_size = len(data) // k
        data_shards = [data[i*shard_size:(i+1)*shard_size] for i in range(k)]
        parity_shards = self._compute_parity(data_shards, m)
        return data_shards + parity_shards  # 12 total shards

    def decode(self, shards, k=8, m=4):
        """Reconstruct original data from any k of k+m shards."""
        available = [s for s in shards if s is not None]
        if len(available) < k:
            raise DataLossError("Not enough shards to reconstruct")
        return self._reconstruct(available, k)

class MultipartUpload:
    """Large objects uploaded in parts, then assembled."""
    def __init__(self):
        self.parts = {}  # part_number -> (etag, location)

    def upload_part(self, part_num, data):
        self.parts[part_num] = (md5(data), placement.store(data))

    def complete(self, bucket, key):
        locs = [self.parts[i][1] for i in sorted(self.parts)]
        return metadata.put_object(bucket, key, locations=locs)

Scaling Considerations

  • Erasure coding (8+4) uses 1.5x storage vs 3x for triple replication
  • Metadata DB sharded by bucket+key hash; data nodes added linearly
  • Background integrity checker scans and repairs degraded shards
  • Lifecycle policies auto-transition objects to cold storage tiers
