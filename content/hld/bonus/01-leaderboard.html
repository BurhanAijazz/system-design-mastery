Design: Leaderboard System

Design a real-time leaderboard that can handle millions of players.

Requirements

  • Update player scores in real-time
  • Get top-K players efficiently
  • Get a player's rank

Core Architecture

# Using Redis Sorted Sets concept
class Leaderboard:
    def __init__(self):
        self._scores = {}  # player_id -> score
        self._sorted = []  # maintained sorted list

    def update_score(self, player_id, score):
        self._scores[player_id] = score
        self._rebuild_sorted()

    def get_top_k(self, k):
        return self._sorted[:k]

    def get_rank(self, player_id):
        for i, (pid, score) in enumerate(self._sorted):
            if pid == player_id:
                return i + 1
        return -1

    def _rebuild_sorted(self):
        self._sorted = sorted(
            self._scores.items(),
            key=lambda x: -x[1]
        )

At Scale

  • Use Redis Sorted Sets (ZADD, ZRANGE, ZRANK) for O(log N) operations
  • Partition by game/region for horizontal scaling
  • Cache top-K results with short TTL
