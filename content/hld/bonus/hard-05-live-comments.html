Design: Live Comments System

Design a real-time live comments system for streaming events and social media posts.

Requirements

  • Post comments and see new comments appear in real-time
  • Support threaded replies (parent-child comment trees)
  • Rate limiting to prevent spam and abuse
  • Scale to millions of concurrent viewers on a single popular stream

High-Level Architecture

  • Comment API — POST /comments (write), GET /comments?stream_id=X (initial load)
  • WebSocket Gateway — Maintains persistent connections for real-time push
  • Fan-Out Service — Publishes new comments to all subscribers of a stream
  • Comment Store — Database with parent_id for threading, indexed by stream_id
  • Rate Limiter — Per-user token bucket to throttle comment frequency
  • Moderation Service — Content filter, toxicity scorer, keyword blocklist

Key Components

import asyncio
from collections import defaultdict

class LiveCommentService:
    def __init__(self):
        self.subscribers = defaultdict(set)  # stream_id -> set of ws
        self.rate_limiters = {}  # user_id -> TokenBucket

    async def post_comment(self, user_id, stream_id, text, parent_id=None):
        if not self._check_rate(user_id):
            raise RateLimitExceeded()
        if not moderation.is_clean(text):
            raise ContentBlocked()
        comment = db.insert({
            "stream_id": stream_id, "user_id": user_id,
            "text": text, "parent_id": parent_id,
            "created_at": now()
        })
        await self._fan_out(stream_id, comment)
        return comment

    async def _fan_out(self, stream_id, comment):
        payload = serialize(comment)
        dead = []
        for ws in self.subscribers[stream_id]:
            try:
                await ws.send(payload)
            except ConnectionClosed:
                dead.append(ws)
        for ws in dead:
            self.subscribers[stream_id].discard(ws)

    def subscribe(self, stream_id, ws):
        self.subscribers[stream_id].add(ws)

    def _check_rate(self, user_id):
        if user_id not in self.rate_limiters:
            self.rate_limiters[user_id] = TokenBucket(5, 1)  # 5 burst, 1/sec
        return self.rate_limiters[user_id].allow_request()

Scaling Considerations

  • Hot streams: shard WebSocket connections across multiple gateway nodes via pub/sub (Redis Pub/Sub or Kafka)
  • Fan-out bottleneck: for millions of viewers, batch and sample comments (show top/random subset)
  • Comment threading: store as adjacency list (parent_id) and build tree client-side
  • Separate read/write paths: writes go to primary DB, reads served from cache/replicas
