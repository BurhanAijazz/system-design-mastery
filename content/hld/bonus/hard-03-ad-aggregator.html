Design: Ad Click Aggregator

Design a system that aggregates ad click events in real-time for billing and analytics.

Requirements

  • Ingest millions of click events per second
  • Real-time aggregation: clicks per ad per minute for billing
  • Fraud detection: filter bots, click farms, duplicate clicks
  • Support historical queries with time-window rollups

High-Level Architecture

  • Click Ingestion (Kafka) — High-throughput event stream from ad servers
  • Stream Processor (Flink/Spark) — Real-time windowed aggregation
  • Fraud Filter — IP rate limiting, device fingerprint dedup, ML scorer
  • Aggregation Store — Pre-computed rollups in OLAP DB (ClickHouse, Druid)
  • Query Service — APIs for dashboards, billing, and advertiser reports

Key Components

from collections import defaultdict
import time

class ClickAggregator:
    def __init__(self, window_sec=60):
        self.window_sec = window_sec
        # (ad_id, window_start) -> count
        self.counts = defaultdict(int)
        self.seen = set()  # dedup: (click_id)

    def process_event(self, event):
        if event["click_id"] in self.seen:
            return  # duplicate
        if self._is_fraud(event):
            return  # filtered
        self.seen.add(event["click_id"])
        window = int(event["ts"] // self.window_sec) * self.window_sec

Scaling Considerations

  • Kafka partitions by ad_id ensure all clicks for an ad go to one aggregator
  • Watermarks handle late-arriving events; allow a grace period before finalizing
  • Lambda architecture: real-time stream layer + batch layer for reconciliation
  • Exactly-once via Kafka transactions + idempotent OLAP upserts
