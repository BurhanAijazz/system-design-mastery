Design: Distributed Job Scheduler

Design a job scheduler that executes recurring and one-off tasks reliably at scale.

Requirements

  • Support cron-like schedules and one-time delayed jobs
  • At-least-once or exactly-once execution guarantees
  • Scalable worker pool with automatic retry and dead letter queue
  • Dashboard for job status, history, and manual re-trigger

High-Level Architecture

  • Scheduler Service — Evaluates cron expressions, enqueues jobs when due
  • Job Queue — Persistent message queue (Kafka / SQS) for pending jobs
  • Worker Pool — Stateless workers that pull and execute jobs
  • Job Store (DB) — Job definitions, schedules, status, execution history
  • Dead Letter Queue — Failed jobs after max retries land here for inspection

Key Components

import time, threading
from datetime import datetime

class JobScheduler:
    def __init__(self):
        self.jobs = {}       # job_id -> JobDef
        self.queue = []      # min-heap by next_run_at

    def register(self, job_id, cron_expr, handler):
        next_run = self._next_cron_time(cron_expr)
        self.jobs[job_id] = {
            "cron": cron_expr, "handler": handler,
            "next_run": next_run, "retries": 0
        }
        heapq.heappush(self.queue, (next_run, job_id))

    def run_loop(self):
        while True:
            now = time.time()
            while self.queue and self.queue[0][0] <= now:
                _, job_id = heapq.heappop(self.queue)
                self._execute(job_id)
            time.sleep(1)

    def _execute(self, job_id):
        job = self.jobs[job_id]
        try:
            job["handler"]()
            job["retries"] = 0
        except Exception:
            job["retries"] += 1
            if job["retries"] > 3:
                self._send_to_dlq(job_id)
                return
        # Reschedule next occurrence
        job["next_run"] = self._next_cron_time(job["cron"])
        heapq.heappush(self.queue, (job["next_run"], job_id))

Scaling Considerations

  • Leader election (via Zookeeper/etcd) ensures only one scheduler fires each job
  • Workers are stateless — scale horizontally based on queue depth
  • Idempotent job handlers enable safe retries for exactly-once semantics
  • Partition jobs by tenant or priority to isolate workloads
