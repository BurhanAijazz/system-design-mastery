Design: API Rate Limiter

Design a rate limiter that enforces per-user request limits across a distributed API fleet.

Requirements

  • Limit each user to N requests per time window
  • Support multiple algorithms: token bucket, sliding window
  • Distributed enforcement across multiple API servers
  • Return 429 Too Many Requests with Retry-After header

High-Level Architecture

  • API Gateway — Intercepts every request before reaching backend
  • Rate Limiter Service — Checks and decrements quota
  • Redis — Shared state for distributed counters and token buckets
  • Rules Engine — Configurable limits per user tier, endpoint, or IP

Key Components

import time

class TokenBucket:
    def __init__(self, capacity, refill_rate):
        self.capacity = capacity
        self.tokens = capacity
        self.refill_rate = refill_rate  # tokens per second
        self.last_refill = time.time()

    def allow_request(self):
        self._refill()
        if self.tokens >= 1:
            self.tokens -= 1
            return True
        return False

    def _refill(self):
        now = time.time()
        elapsed = now - self.last_refill
        self.tokens = min(
            self.capacity,
            self.tokens + elapsed * self.refill_rate
        )
        self.last_refill = now

Scaling Considerations

  • Redis atomic Lua scripts prevent race conditions across API nodes
  • Local in-memory cache per server reduces Redis round-trips (sync periodically)
  • Sliding window log is more accurate but uses more memory than fixed window
  • Use consistent hashing so each user maps to one Redis shard
