Design: Metrics Monitoring

Design a metrics monitoring system that collects, stores, and alerts on time-series data.

Requirements

  • Collect metrics (CPU, memory, request latency, error rate) from thousands of hosts
  • Store time-series data with configurable retention and downsampling
  • Query and visualize metrics with flexible aggregation (avg, p99, sum)
  • Alerting rules that trigger notifications when thresholds are breached

High-Level Architecture

  • Collection Agent — Runs on each host; exposes /metrics endpoint (pull model)
  • Scraper / Ingestion — Prometheus-like scraper pulls metrics at intervals
  • Time-Series DB — Stores (metric_name, labels, timestamp, value) efficiently
  • Query Engine — PromQL-like language for aggregation and filtering
  • Alert Manager — Evaluates rules periodically, deduplicates, routes alerts

Key Components

import time
from collections import defaultdict

class TimeSeriesDB:
    def __init__(self):
        # metric_key -> list of (timestamp, value)
        self.series = defaultdict(list)

    def write(self, metric, labels, value):
        key = f"{metric}|{sorted(labels.items())}"
        self.series[key].append((time.time(), value))

    def query(self, metric, labels, start, end, agg="avg"):
        key = f"{metric}|{sorted(labels.items())}"
        points = [
            v for t, v in self.series[key]
            if start <= t <= end
        ]
        if agg == "avg":
            return sum(points) / len(points) if points else 0
        if agg == "max":
            return max(points) if points else 0
        if agg == "p99":
            points.sort()
            idx = int(len(points) * 0.99)
            return points[idx] if points else 0

class AlertRule:
    def __init__(self, name, query, threshold, operator="gt"):
        self.name = name
        self.query = query        # e.g., ("cpu_usage", {"host":"*"})
        self.threshold = threshold
        self.operator = operator   # "gt" or "lt"

    def evaluate(self, tsdb):
        value = tsdb.query(*self.query, time.time()-300, time.time())
        if self.operator == "gt" and value > self.threshold:
            return f"ALERT {self.name}: {value} > {self.threshold}"
        return None

Scaling Considerations

  • Downsample old data (1s → 1m → 1h) to reduce storage
  • Shard time-series by metric name hash across TSDB nodes
  • Pull model (Prometheus) is simpler; push model (StatsD) suits short-lived jobs
  • Alert deduplication and grouping prevent notification storms
