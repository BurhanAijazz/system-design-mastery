Design: Unique ID Generator

Design a distributed unique ID generation system.

Approaches

  • UUID: 128-bit, no coordination needed, but not sortable
  • Snowflake ID: 64-bit, time-ordered, used by Twitter
  • Database auto-increment: Simple but doesn't scale horizontally

Snowflake Implementation

import time

class SnowflakeGenerator:
    EPOCH = 1609459200000  # Jan 1, 2021

    def __init__(self, machine_id, datacenter_id):
        self.machine_id = machine_id & 0x1F      # 5 bits
        self.datacenter_id = datacenter_id & 0x1F # 5 bits
        self.sequence = 0
        self.last_timestamp = -1

    def generate(self):
        timestamp = int(time.time() * 1000) - self.EPOCH
        if timestamp == self.last_timestamp:
            self.sequence = (self.sequence + 1) & 0xFFF  # 12 bits
            if self.sequence == 0:
                timestamp = self._wait_next_millis()
        else:
            self.sequence = 0

        self.last_timestamp = timestamp

        return (
            (timestamp << 22) |
            (self.datacenter_id << 17) |
            (self.machine_id << 12) |
            self.sequence
        )

    def _wait_next_millis(self):
        ts = int(time.time() * 1000) - self.EPOCH
        while ts <= self.last_timestamp:
            ts = int(time.time() * 1000) - self.EPOCH
        return ts

Key Takeaways

  • Snowflake: 41 bits timestamp + 5 bits datacenter + 5 bits machine + 12 bits sequence
  • ~4096 IDs per millisecond per machine
  • Time-ordered for easy sorting and partitioning
