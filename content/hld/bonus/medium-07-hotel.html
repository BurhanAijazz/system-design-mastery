Design: Hotel Reservation

Design a hotel reservation system that handles availability, booking, and pricing.

Requirements

  • Search available rooms by date range, location, and room type
  • Book a room with double-booking prevention
  • Support overbooking up to a configurable percentage
  • Dynamic pricing based on demand, season, and occupancy
  • Cancellation with refund policy enforcement

High-Level Architecture

  • Search Service — Queries inventory index for available rooms
  • Booking Service — Reserves rooms transactionally, prevents double-book
  • Inventory Service — Availability calendar: room_type + date → remaining count
  • Pricing Service — Computes nightly rate from demand signals
  • Notification Service — Confirmation emails, cancellation alerts

Key Components

class InventoryService:
    def __init__(self, db):
        self.db = db

    def check_availability(self, hotel_id, room_type, check_in, check_out):
        """Returns min available rooms across the date range."""
        dates = date_range(check_in, check_out)
        counts = [
            self.db.get_available(hotel_id, room_type, d)
            for d in dates
        ]
        return min(counts)

    def reserve(self, hotel_id, room_type, check_in, check_out):
        """Atomic reservation using optimistic locking."""
        with self.db.transaction():
            for d in date_range(check_in, check_out):
                row = self.db.select_for_update(hotel_id, room_type, d)
                overbooking_limit = row["total"] * 1.1  # 10% overbook
                if row["booked"] >= overbooking_limit:
                    raise NoAvailabilityError(d)
                self.db.increment_booked(hotel_id, room_type, d)
            return self.db.create_reservation(
                hotel_id, room_type, check_in, check_out
            )

class PricingService:
    def compute_rate(self, base_rate, occupancy_pct, is_weekend):
        multiplier = 1.0
        if occupancy_pct > 0.8:
            multiplier += 0.3  # surge pricing
        if is_weekend:
            multiplier += 0.15
        return round(base_rate * multiplier, 2)

Scaling Considerations

  • Pessimistic locking (SELECT FOR UPDATE) prevents double-booking per date-room
  • Cache search results with short TTL; invalidate on booking
  • Shard inventory DB by hotel_id or region
  • Async cancellation processing to release inventory and trigger refund
