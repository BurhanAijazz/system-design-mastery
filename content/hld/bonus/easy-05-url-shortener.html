Design: URL Shortener (Basic)

Design a URL shortening service that maps long URLs to short, unique aliases.

Requirements

  • Generate a short URL given a long URL
  • Redirect short URL to original with 301 or 302
  • Handle high read-to-write ratio (100:1 typical)
  • Short URLs should be compact (6-8 characters)

High-Level Architecture

  • API Service — POST /shorten, GET /{shortCode} redirect
  • ID Generator — Auto-increment counter or distributed snowflake IDs
  • Encoder — Base62 encode the numeric ID into a short string
  • Database — Mapping table: short_code → long_url, created_at, user_id
  • Cache (Redis) — Hot short codes cached for fast redirect

Key Components

import string

ALPHABET = string.digits + string.ascii_letters  # 62 chars

def base62_encode(num):
    if num == 0:
        return ALPHABET[0]
    result = []
    while num > 0:
        result.append(ALPHABET[num % 62])
        num //= 62
    return ''.join(reversed(result))

def shorten(long_url, counter):

Scaling Considerations

  • 301 (permanent) lets browsers cache; 302 (temporary) routes through server for analytics
  • Pre-allocate ID ranges per server to avoid single-counter bottleneck
  • Read-heavy: put a CDN or cache in front of redirect path
  • Database sharding by short_code hash for horizontal scale
