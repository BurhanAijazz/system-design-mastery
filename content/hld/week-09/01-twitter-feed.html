Twitter / Newsfeed Design

The Twitter newsfeed is the most classic social system design question. The core challenge is the fan-out problem: when a user tweets, how do millions of followers see it in their timeline?

Define: Requirements

  • Functional: Post tweets (280 chars + optional media), follow/unfollow users, home timeline (feed of followed users' tweets), user profile timeline, search, likes, retweets, replies
  • Non-Functional: Timeline loads in <200ms, tweets appear in followers' feeds within seconds, high availability, eventual consistency acceptable
  • Out of Scope: Direct messages, trending topics, ads, recommendations

Estimate

# 300M DAU, each follows ~200 people
# 10% tweet daily, avg 2 tweets/day

# WRITES:
#   300M x 10% x 2 = 60M tweets/day
#   60M / 86400 ~ 700 tweets/sec (peak ~2,100/sec)

# READS:
#   300M x 10 timeline loads = 3B/day
#   3B / 86400 ~ 35,000 QPS (peak ~105,000 QPS)
# READ-HEAVY: 50:1 read-to-write ratio

# TIMELINE CACHE:
#   Each user: last 200 tweet IDs = 1.6 KB
#   300M x 1.6 KB = 480 GB (fits in Redis cluster)

The Fan-Out Problem

This is the heart of the design. There are two approaches:

# APPROACH 1: Fan-Out on Write (Push Model)
# When user tweets -> write to ALL followers' timelines
# Pros: Timeline reads are instant (pre-computed)
# Cons: Celebrity problem - user with 50M followers
#        triggers 50M cache writes per tweet!

# APPROACH 2: Fan-Out on Read (Pull Model)
# When user loads timeline -> fetch tweets from all followed users
# Pros: No write amplification
# Cons: Slow reads - must query 200 users' tweets and merge

# APPROACH 3: Hybrid (Twitter's actual approach)
# Regular users: fan-out on write (push to followers)
# Celebrities (>10K followers): fan-out on read
# Timeline = cached_tweets UNION celebrity_tweets

Architecture

  • Tweet Service: Handles tweet creation, stores in database
  • Fan-out Service: Pushes tweet IDs to followers' timeline caches (Redis sorted sets)
  • Timeline Service: Reads pre-computed timeline from cache, merges with celebrity tweets on-the-fly
  • Social Graph Service: Manages follow relationships (60B edges for 300M users)
  • Search Service: Elasticsearch index on tweet text and metadata

Key Design Decisions

  • Celebrity threshold: Users with >10K followers use pull model; below that use push model
  • Timeline cache: Redis sorted sets with tweet timestamp as score, trimmed to last 200 entries
  • Media storage: Images and videos on S3 behind CDN; only URLs stored in tweet metadata
  • Eventual consistency: A tweet may take a few seconds to appear in all followers' feeds, which is acceptable
