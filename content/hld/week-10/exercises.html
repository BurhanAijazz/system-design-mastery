Week 10 Exercises: Media and Streaming

These exercises explore video transcoding pipelines, streaming architectures, and file synchronization covered this week.

Exercise 1: Video Transcoding Priority Queue

Design the transcoding pipeline for a YouTube-like platform that prioritizes certain jobs over others.

  • New uploads from verified channels should be transcoded before unverified uploads
  • Lower resolutions (240p, 360p) should complete first so users can start watching sooner
  • Re-transcoding jobs (e.g., new codec support) should never block new uploads

Tasks:

  • Design the priority queue with at least 3 priority levels
  • How many transcoding workers are needed for 500 hours of video uploaded per minute?
  • Estimate transcoding time: a 10-minute 1080p video takes approximately 4x real-time on a single worker. How do you parallelize?
  • What happens when a transcoding worker crashes mid-job? Design the recovery mechanism.

Exercise 3: File Sync Conflict Simulator

Implement and test different conflict resolution strategies for a Google Drive-like system.

# Scenario: 3 devices editing the same file
# Device A: online, edits paragraph 2
# Device B: offline for 1 hour, edits paragraph 2 and 5
# Device C: online, edits paragraph 5
# Device B comes back online. What happens?

# Strategies to implement and compare:
# 1. Last-write-wins (simple but loses data)
# 2. Conflict copy (Google Drive approach)
# 3. Block-level merge (merge non-overlapping changes)
# 4. Three-way merge (diff3 against common ancestor)

Tasks:

  • Implement at least two strategies and simulate the scenario above
  • Measure data loss for each strategy
  • Which strategy would you use for binary files (images, PDFs) vs text files?
  • How does block-level chunking affect conflict granularity?
