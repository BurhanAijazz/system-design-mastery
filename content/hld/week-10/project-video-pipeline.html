Project: Video Upload Pipeline Simulator

This project simulates the full video upload and transcoding pipeline: chunked resumable uploads, parallel transcoding to multiple resolutions, HLS manifest generation, CDN caching, adaptive bitrate streaming, and view counting.

Core Data Structures

from dataclasses import dataclass, field
from typing import Dict, List

@dataclass
class VideoChunk:
    chunk_id: int
    offset_bytes: int
    size_bytes: int
    uploaded: bool = False
    upload_time_ms: float = 0.0

@dataclass
class TranscodeJob:
    job_id: int
    video_id: str
    resolution: str
    status: str = "queued"  # queued, processing, completed, failed
    progress_pct: float = 0.0
    output_size_mb: float = 0.0
    worker_id: int = -1

@dataclass
class Video:
    video_id: str
    title: str
    duration_sec: float
    raw_size_mb: float
    status: str = "uploading"  # uploading, transcoding, ready
    chunks: List[VideoChunk] = field(default_factory=list)
    transcode_jobs: List[TranscodeJob] = field(default_factory=list)
    resolutions_ready: Dict[str, float] = field(default_factory=dict)
    view_count: int = 0

Upload Service

The upload service handles chunked, resumable uploads. Large files are split into 5 MB chunks, allowing the client to resume from where it left off after a network failure.

class UploadService:
    CHUNK_SIZE_MB = 5  # 5 MB per chunk

    def start_upload(self, video_id, title, raw_size_mb, duration_sec):
        num_chunks = math.ceil(raw_size_mb / self.CHUNK_SIZE_MB)
        chunks = [VideoChunk(i, i * self.CHUNK_SIZE_MB * 1e6,
                             self.CHUNK_SIZE_MB * 1e6)
                  for i in range(num_chunks)]
        video = Video(video_id, title, duration_sec,
                      raw_size_mb, chunks=chunks)
        return video

    def upload_chunk(self, video, chunk_id):
        chunk = video.chunks[chunk_id]
        chunk.uploaded = True
        # Check if all chunks complete
        if all(c.uploaded for c in video.chunks):
            video.status = "transcoding"

Transcoding Pipeline

  • Once upload completes, transcoding jobs are created for each target resolution
  • Jobs run in parallel across a pool of workers
  • Lower resolutions complete first, enabling early playback
  • Each completed resolution generates HLS segments and updates the manifest

Adaptive Bitrate Streaming

  • Client measures available bandwidth and requests the appropriate resolution
  • If bandwidth drops mid-playback, client seamlessly switches to a lower resolution
  • CDN caches popular segments at edge locations for low-latency delivery

Key Takeaways

  • Chunked upload with resumability is essential for large files over unreliable networks
  • Parallel transcoding to multiple resolutions enables progressive availability
  • HLS manifests are the glue between transcoded segments and the adaptive streaming client
  • View counting uses batch aggregation to avoid hot partition problems on viral videos
