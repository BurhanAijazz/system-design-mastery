Command Pattern

The Command pattern encapsulates a request as an object, allowing you to parameterize clients with different requests, queue requests, log them, and support undo/redo operations.

The Problem: Direct Coupling

When button clicks directly call methods on an editor, there is no way to undo actions, record history, or support macros:

class TextEditorNaive:
    """Editor where button clicks directly call methods.
    No undo, no history, no macro support."""

    def __init__(self):
        self.content = ""

    def type_text(self, text: str) -> None:
        self.content += text

    def delete_last(self, count: int = 1) -> None:
        self.content = self.content[:-count]

Once text is deleted, the history is lost. There is no way to undo or replay actions.

Key Components

  • Command (interface) -- declares execute() and undo()
  • ConcreteCommand -- implements execute/undo, holds a reference to the receiver
  • Receiver -- the object that actually performs the work
  • Invoker -- triggers commands without knowing what they do

The Receiver

class TextDocument:
    """The receiver -- performs the actual text operations."""

    def __init__(self):
        self.content = ""

    def insert(self, text: str, position: int = -1) -> None:
        if position == -1:
            position = len(self.content)
        self.content = self.content[:position] + text + self.content[position:]

    def delete(self, position: int, count: int) -> str:
        deleted = self.content[position:position + count]
        self.content = self.content[:position] + self.content[position + count:]
        return deleted

Command Interface and Concrete Commands

from abc import ABC, abstractmethod

class Command(ABC):
    """Base command with execute and undo."""

    @abstractmethod
    def execute(self) -> None:
        pass

    @abstractmethod
    def undo(self) -> None:
        pass


class TypeCommand(Command):
    """Command to type text at a position."""

    def __init__(self, document: TextDocument, text: str, position: int = -1):
        self._doc = document
        self._text = text
        self._position = position if position != -1 else len(document.content)

    def execute(self) -> None:
        self._doc.insert(self._text, self._position)

    def undo(self) -> None:
        self._doc.delete(self._position, len(self._text))


class DeleteCommand(Command):
    """Command to delete text -- stores deleted text for undo."""

    def __init__(self, document: TextDocument, position: int, count: int):
        self._doc = document
        self._position = position
        self._count = count
        self._deleted_text = ""  # saved on execute for undo

    def execute(self) -> None:
        self._deleted_text = self._doc.delete(self._position, self._count)

    def undo(self) -> None:
        self._doc.insert(self._deleted_text, self._position)

The Invoker (Command History)

class CommandHistory:
    """Invoker -- maintains undo/redo stacks."""

    def __init__(self):
        self._undo_stack: list[Command] = []
        self._redo_stack: list[Command] = []

    def execute(self, command: Command) -> None:
        command.execute()
        self._undo_stack.append(command)
        self._redo_stack.clear()  # new action invalidates redo

    def undo(self) -> bool:
        if not self._undo_stack:
            return False
        command = self._undo_stack.pop()
        command.undo()
        self._redo_stack.append(command)
        return True

    def redo(self) -> bool:
        if not self._redo_stack:
            return False
        command = self._redo_stack.pop()
        command.execute()
        self._undo_stack.append(command)
        return True

Usage Example

doc = TextDocument()
history = CommandHistory()

# Type "Hello World"
history.execute(TypeCommand(doc, "Hello "))
history.execute(TypeCommand(doc, "World"))
print(doc.content)  # "Hello World"

# Undo last type
history.undo()
print(doc.content)  # "Hello "

# Redo
history.redo()
print(doc.content)  # "Hello World"

When to Use the Command Pattern

  • Undo/Redo -- text editors, drawing tools, form builders
  • Transaction logging -- record and replay operations
  • Macro recording -- save a sequence of commands as a macro
  • Task queues -- enqueue commands for later execution
  • Remote execution -- serialize commands and send over a network

Key Design Insight

The Command pattern turns method calls into objects. This lets you store, queue, and reverse operations. The invoker (button, menu, scheduler) never knows what work the command performs -- it just calls execute().
