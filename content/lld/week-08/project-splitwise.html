Project: Splitwise

A full LLD implementation of a Splitwise-like expense sharing platform. This project combines multiple design patterns: Strategy for split calculation, Observer for group notifications, State for expense lifecycle, and Command for undo support.

Requirements

Functional:

  • Create users with name and email
  • Create groups with multiple members
  • Add expenses with different split types (equal, exact, percentage, share-based)
  • Track who paid and who owes
  • View balances per user (net amount owed/owing)
  • Settle debts between two users
  • Simplify debts (minimize number of transactions)
  • View expense history for a group
  • Undo the last expense added

Out of scope: Currency conversion, recurring expenses, image/receipt upload.

Architecture

The system is organized around an ExpenseManager that coordinates Groups, a Ledger, and SplitStrategies:

# Core entities
class ExpenseStatus(Enum):
    PENDING = "pending"
    SETTLED = "settled"
    CANCELLED = "cancelled"

class SplitType(Enum):
    EQUAL = "equal"
    EXACT = "exact"
    PERCENTAGE = "percentage"
    SHARE = "share"

@dataclass
class User:
    user_id: str
    name: str
    email: str

@dataclass
class Split:
    """How much a single user owes in an expense."""
    user_id: str
    amount: float

Design Patterns Used

  • Strategy -- Split calculation: EqualSplit divides evenly (handling rounding), ExactSplit uses specified amounts, PercentageSplit computes from percentages, ShareSplit divides proportionally
  • Observer -- When an expense is added to a group, all group members are notified
  • State -- Expenses transition through PENDING -> SETTLED or CANCELLED
  • Command -- AddExpense command supports undo by cancelling the expense and reversing ledger entries

Key Implementation Details

class EqualSplit(SplitStrategy):
    def calculate(self, total, participants, **kwargs):
        share = round(total / len(participants), 2)
        splits = [Split(uid, share) for uid in participants]
        # Handle rounding remainder
        diff = round(total - share * len(participants), 2)
        if diff != 0:
            splits[0].amount += diff
        return splits

Implementation Steps

  • Define entities: User, Split, Expense, Group
  • Implement split strategies (Strategy pattern)
  • Build the append-only Ledger for balance tracking
  • Create the ExpenseManager service that coordinates groups, expenses, and the ledger
  • Add Observer notifications for group members
  • Implement debt simplification using net balance calculation
  • Add Command pattern for undo support
