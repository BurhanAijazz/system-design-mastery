Week 8 Exercises: Platform Systems

These exercises reinforce the platform system patterns covered this week: user and role management, search and catalog design, notification dispatch, and transaction ledgers. Work through each exercise, focusing on identifying the right entities and applying the appropriate design patterns.

Exercise 1: Permission System for a Content Platform

Design a role-based permission system for a content platform (like Medium or Dev.to) with the following requirements:

  • Three roles: READER (can read articles), AUTHOR (can read and write articles), ADMIN (full access including delete and user management)
  • Permissions should be composable using Flag enums (READ, WRITE, DELETE, MANAGE_USERS)
  • A user can have multiple roles; effective permissions are the union of all role permissions
  • Implement a has_permission(user, permission) check that aggregates across roles

Key patterns: Flag enum for permissions, Set of roles per user, aggregate root (User).

# Starter hint:
class Permission(Flag):
    READ = auto()
    WRITE = auto()
    DELETE = auto()
    MANAGE_USERS = auto()

# Implement User with a set of roles
# Implement has_permission() that unions all role permissions

Exercise 2: Multi-Criteria Search with Pagination

Build a search system for a hotel listing platform. Given a list of hotels, implement:

  • A SearchCriteria data class with fields: city, min_price, max_price, min_rating, amenities (list), sort_by
  • A filter chain that applies each non-None criterion sequentially (Chain of Responsibility)
  • At least two sort strategies: by price (ascending) and by rating (descending)
  • A generic Page class that wraps results with page number, total pages, and has_next

Key patterns: Chain of Responsibility for filters, Strategy for sorting, Generic for pagination.

# Starter hint:
@dataclass
class SearchCriteria:
    city: Optional[str] = None
    min_price: Optional[float] = None
    max_price: Optional[float] = None
    min_rating: Optional[float] = None
    sort_by: str = "price"
    page: int = 1
    page_size: int = 10

# Implement filter chain, sort strategies, and Page wrapper
