Custom Iterator Pattern

The Iterator pattern provides a way to access elements of a collection sequentially without exposing its underlying representation. Beyond Python's built-in __iter__/__next__ protocol, the design-pattern-level iterator supports multiple traversal strategies over the same collection.

Key Insight

A single collection can support multiple traversal strategies. A binary tree can be iterated in-order, pre-order, post-order, or level-order. A playlist can be iterated sequentially, shuffled, or on repeat.

Tree Traversal Iterators

from abc import ABC, abstractmethod
from typing import Optional, Iterator, List
from collections import deque

class TreeNode:
    """A node in a binary tree."""
    def __init__(self, value: int,
                 left: Optional["TreeNode"] = None,
                 right: Optional["TreeNode"] = None):
        self.value = value
        self.left = left
        self.right = right


class TreeIterator(ABC):
    """Abstract iterator over a binary tree."""
    def __init__(self, root: Optional[TreeNode]):
        self._root = root

    @abstractmethod
    def __iter__(self) -> Iterator[int]:
        pass

In-Order Iterator (Left, Root, Right)

Gives sorted order for Binary Search Trees:

class InOrderIterator(TreeIterator):
    """Left -> Root -> Right. Gives sorted order for BSTs."""
    def __iter__(self) -> Iterator[int]:
        stack: List[TreeNode] = []
        current = self._root
        while stack or current:
            while current:
                stack.append(current)
                current = current.left
            current = stack.pop()
            yield current.value
            current = current.right

Pre-Order Iterator (Root, Left, Right)

class PreOrderIterator(TreeIterator):
    """Root -> Left -> Right. Useful for copying trees."""
    def __iter__(self) -> Iterator[int]:
        if not self._root:
            return
        stack = [self._root]
        while stack:
            node = stack.pop()
            yield node.value
            if node.right:
                stack.append(node.right)
            if node.left:
                stack.append(node.left)

Post-Order Iterator (Left, Right, Root)

class PostOrderIterator(TreeIterator):
    """Left -> Right -> Root. Useful for deletion."""
    def __iter__(self) -> Iterator[int]:
        if not self._root:
            return
        stack1 = [self._root]
        stack2: List[TreeNode] = []
        while stack1:
            node = stack1.pop()
            stack2.append(node)
            if node.left:
                stack1.append(node.left)
            if node.right:
                stack1.append(node.right)
        while stack2:
            yield stack2.pop().value

Usage: Same Tree, Multiple Traversals

# Build tree:     4
#                / \
#               2   6
#              / \
#             1   3

root = TreeNode(4,
    TreeNode(2, TreeNode(1), TreeNode(3)),
    TreeNode(6)
)

print(list(InOrderIterator(root)))    # [1, 2, 3, 4, 6]
print(list(PreOrderIterator(root)))   # [4, 2, 1, 3, 6]
print(list(PostOrderIterator(root)))  # [1, 3, 2, 6, 4]

Iterable Collection Pattern

The collection itself can provide different iterators through named methods:

class BinaryTree:
    def __init__(self, root: Optional[TreeNode] = None):
        self._root = root

    def in_order(self) -> InOrderIterator:
        return InOrderIterator(self._root)

    def pre_order(self) -> PreOrderIterator:
        return PreOrderIterator(self._root)

    def post_order(self) -> PostOrderIterator:
        return PostOrderIterator(self._root)

    def __iter__(self):
        """Default iteration is in-order."""
        return iter(InOrderIterator(self._root))

When to Use Custom Iterators

  • Trees and graphs -- DFS, BFS, in-order, level-order traversals
  • Playlists -- sequential, shuffled, repeat, filtered
  • Paginated results -- lazy-loading pages from an API
  • File systems -- recursive directory traversal with filtering
  • Database cursors -- streaming rows without loading all into memory

Iterator vs Generator

In Python, generators (yield) are the idiomatic way to implement iterators. The design pattern focuses on providing multiple traversal strategies over a single data structure, which generators support elegantly.
