Memento Pattern

The Memento pattern captures and restores an object's internal state without violating encapsulation. The object itself creates the snapshot; an external caretaker stores and manages snapshots.

Key Components

  • Originator -- the object whose state is saved/restored
  • Memento -- an opaque snapshot of the originator's state
  • Caretaker -- stores mementos (undo stack, history, save slots)

The Memento is a "black box" -- only the originator can read/write it. The caretaker just stores and passes it around.

The Problem: Exposing Internal State

class EditorNaive:
    """To support undo, we'd need to expose all internal state
    or clone the entire object. Both approaches are messy."""

    def __init__(self):
        self.content = ""
        self.cursor_pos = 0
        self.selection_start = 0
        self.selection_end = 0
        self.font = "Arial"
        self.font_size = 12

    def type_text(self, text: str) -> None:
        self.content = (self.content[:self.cursor_pos]
                       + text
                       + self.content[self.cursor_pos:])
        self.cursor_pos += len(text)

To undo, external code would need to save ALL those fields. That breaks encapsulation -- external code knows about internal state.

Solution: Memento Pattern

from dataclasses import dataclass

@dataclass(frozen=True)
class EditorMemento:
    """Memento -- an immutable snapshot of the editor's state.
    frozen=True makes it immutable (can't accidentally modify it)."""
    content: str
    cursor_pos: int
    font: str
    font_size: int
    timestamp: str = ""


class TextEditor:
    """Originator -- creates and restores from mementos."""

    def __init__(self):
        self._content = ""
        self._cursor_pos = 0
        self._font = "Arial"
        self._font_size = 12

    def type_text(self, text: str) -> None:
        self._content = (self._content[:self._cursor_pos]
                        + text
                        + self._content[self._cursor_pos:])
        self._cursor_pos += len(text)

    def delete(self, count: int = 1) -> None:
        start = max(0, self._cursor_pos - count)
        self._content = self._content[:start] + self._content[self._cursor_pos:]
        self._cursor_pos = start

    # --- Memento methods ---

    def save(self) -> EditorMemento:
        """Create a snapshot of current state."""
        return EditorMemento(
            content=self._content,
            cursor_pos=self._cursor_pos,
            font=self._font,
            font_size=self._font_size,
        )

    def restore(self, memento: EditorMemento) -> None:
        """Restore state from a snapshot."""
        self._content = memento.content
        self._cursor_pos = memento.cursor_pos
        self._font = memento.font
        self._font_size = memento.font_size

The Caretaker

class EditorHistory:
    """Caretaker -- stores mementos without knowing their contents."""

    def __init__(self, editor: TextEditor):
        self._editor = editor
        self._undo_stack: list[EditorMemento] = []
        self._redo_stack: list[EditorMemento] = []

    def save(self) -> None:
        """Save current state before each action."""
        self._undo_stack.append(self._editor.save())
        self._redo_stack.clear()

    def undo(self) -> bool:
        if not self._undo_stack:
            return False
        self._redo_stack.append(self._editor.save())
        memento = self._undo_stack.pop()
        self._editor.restore(memento)
        return True

    def redo(self) -> bool:
        if not self._redo_stack:
            return False
        self._undo_stack.append(self._editor.save())
        memento = self._redo_stack.pop()
        self._editor.restore(memento)
        return True

Usage

editor = TextEditor()
history = EditorHistory(editor)

history.save()
editor.type_text("Hello ")
history.save()
editor.type_text("World")

# Content is "Hello World"
history.undo()   # Back to "Hello "
history.undo()   # Back to ""
history.redo()   # Forward to "Hello "

Memento vs Command for Undo

  • Command stores the inverse operation -- works well when operations are easily reversible
  • Memento stores the full state snapshot -- works when state is complex or operations are hard to reverse
  • Both patterns are often combined: Command for undo logic, Memento for cursor/selection state

When to Use

  • Text editors -- save/restore content and cursor state
  • Game save/load -- snapshot game state into save slots
  • Transaction rollback -- restore state if a transaction fails
  • Configuration snapshots -- revert settings to a previous version
