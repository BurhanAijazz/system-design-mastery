Traffic Signal System Walkthrough

This lesson walks through designing a Traffic Signal System using the 6-step LLD framework. It combines the State pattern for signal transitions, Observer for intersection coordination, and Mediator for multi-signal synchronization.

Step 1: Clarify Requirements

  • "Are we designing a single intersection or multiple?" -- Start with one, then discuss coordination
  • "Standard 4-way intersection with traffic lights?" -- Yes, include pedestrian crossing signals too
  • "Should I handle emergency vehicle preemption?" -- Yes, that is a great edge case

Functional Requirements:

  • Traffic signal with Red, Yellow, Green states
  • Timed transitions (Green 30s -> Yellow 5s -> Red 30s)
  • Coordinate signals at a 4-way intersection (when N-S is Green, E-W must be Red)
  • Pedestrian crossing signal
  • Emergency vehicle override (all Red except emergency direction)
  • Manual override by traffic controller
  • Configurable timing for peak/off-peak hours

Out of scope: Turn signals, sensor-triggered signals, multi-intersection corridor synchronization.

Step 2: Identify Entities

# Core entities:
# TrafficSignal     - id, direction, state, timer config
# SignalState       - Red, Yellow, Green, PedestrianWalk
# Intersection      - id, signals (N,S,E,W), controller
# Road              - id, name, direction, signal
# TrafficController - intersection, schedule, mode
# TimerConfig       - green_duration, yellow_duration,
#                     red_duration, pedestrian_duration
# EmergencyOverride - direction, duration, priority

Signal State Machine

Each signal cycles through states with configurable timing:

  • Green (30s) -> Yellow (5s) -> Red (30s) -> Green (repeat)
  • Emergency override: all signals go Red except the emergency direction
  • Pedestrian phase: inserted during the Red phase of vehicle signals

Intersection Coordination

The intersection cycles through phases to ensure safety:

  • Phase 1: N-S Green, E-W Red
  • Phase 2: N-S Yellow, E-W Red
  • Phase 3: N-S Red, E-W Green
  • Phase 4: N-S Red, E-W Yellow
  • (repeat)

Step 3: Design Patterns

  • State -- Each signal state (RedState, YellowState, GreenState) handles its own transition logic and timer. The signal delegates behavior to its current state object
  • Observer -- When one signal changes state, the intersection controller is notified and can coordinate other signals accordingly
  • Mediator -- The IntersectionController acts as a mediator between signals. Signals do not communicate directly; they go through the controller, which enforces safety invariants (never two conflicting greens)
  • Strategy -- Timing strategies: PeakHourTiming (shorter cycles, more throughput), OffPeakTiming (longer cycles, pedestrian-friendly), EmergencyTiming (immediate override)

Emergency Override Flow

# When an emergency vehicle is detected:
# 1. Controller receives emergency signal with direction
# 2. All signals immediately transition to Red
# 3. Emergency direction signal transitions to Green
# 4. After emergency passes, resume normal cycle

class IntersectionController:
    def handle_emergency(self, direction: Direction):
        for signal in self._signals.values():
            signal.force_red()
        self._signals[direction].force_green()
        # Set timer to resume normal operation

Key Takeaway

The traffic signal system demonstrates how State, Observer, and Mediator patterns work together. State handles individual signal behavior, Observer enables reactive coordination, and Mediator ensures safety by centralizing cross-signal communication. In an interview, emphasize the safety invariant: conflicting directions must never both be Green.
