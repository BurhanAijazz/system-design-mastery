Project: Food Delivery System

A full LLD implementation of a food delivery platform (like Swiggy/Zomato). This project uses State for order lifecycle, Strategy for delivery agent assignment, Observer for real-time order tracking, Chain of Responsibility for order validation, Builder for order construction, and Command for order modifications.

Requirements

Functional:

  • Browse restaurants and menus
  • Add items to cart, place order
  • Order lifecycle with state transitions (Placed -> Confirmed -> Preparing -> ReadyForPickup -> OutForDelivery -> Delivered)
  • Assign delivery agent to order
  • Real-time order status tracking
  • Rate restaurant and agent
  • Order history

Out of scope: Payment gateway, coupons, surge pricing, route optimization.

@dataclass
class Location:
    lat: float
    lng: float

    def distance_to(self, other: Location) -> float:
        return math.sqrt((self.lat - other.lat)**2 +
                         (self.lng - other.lng)**2)

@dataclass
class MenuItem:
    item_id: str
    name: str
    price: float
    category: str
    available: bool = True

Design Patterns Used

  • State -- Each order state is a separate class. PlacedState allows confirm/cancel; PreparingState allows markReady; OutForDeliveryState allows markDelivered. Invalid transitions raise errors instead of silently failing
  • Strategy -- Agent assignment: NearestAgentStrategy finds the closest available agent by location distance; LeastBusyStrategy finds the agent with the fewest active orders. Both implement the same AssignmentStrategy interface
  • Observer -- OrderTracker observes order state changes. When the order transitions, all registered observers (customer app, restaurant dashboard, agent app) receive the update
  • Chain of Responsibility -- Order validation before placement: RestaurantOpenHandler -> ItemsAvailableHandler -> PaymentValidHandler. Each handler checks one condition and forwards if valid
  • Builder -- OrderBuilder provides a fluent API to construct orders: builder.customer(user).restaurant(r).add_item(item, qty).build()
  • Command -- Order modifications (add item, remove item) are commands that support undo

Implementation Steps

  • Define entities: User, Restaurant, Menu, MenuItem, Order, OrderItem, DeliveryAgent, Location
  • Implement the order state machine (State pattern) with all 7 states
  • Build agent assignment strategies (nearest, least busy)
  • Create the OrderService that orchestrates browsing, ordering, tracking
  • Add Observer for real-time order status notifications
  • Implement the validation chain for order placement
