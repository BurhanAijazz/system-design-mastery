Week 10 Exercises: Advanced Problems and Mock Interviews

These exercises simulate real LLD interview scenarios. For each exercise, follow the 5-5-5-25-5 framework: clarify requirements, identify entities, define interfaces and patterns, implement core logic, and discuss trade-offs. Time yourself to build interview stamina.

Exercise 1: Ride-Sharing System (45 minutes)

Design a ride-sharing system (like Uber/Lyft). Practice the full 6-step framework:

  • Requirements: Riders request rides, drivers accept. System matches rider to nearest available driver. Ride goes through lifecycle: Requested -> Matched -> InProgress -> Completed. Track fare based on distance and time
  • Entities to identify: Rider, Driver, Ride, Location, FareCalculator, RideStatus
  • Patterns to consider: State (ride lifecycle), Strategy (matching algorithm and fare calculation), Observer (ride status updates)

Focus area: State pattern for ride lifecycle and Strategy for driver matching. Write the state machine first, then implement the matching strategy.

# Starter hint:
class RideStatus(Enum):
    REQUESTED = "requested"
    MATCHED = "matched"
    DRIVER_EN_ROUTE = "driver_en_route"
    IN_PROGRESS = "in_progress"
    COMPLETED = "completed"
    CANCELLED = "cancelled"

# then build MatchingStrategy (Nearest, HighestRated)

Exercise 2: Vending Machine (30 minutes)

Design a vending machine -- a classic State pattern problem. Practice completing a simpler design in 30 minutes:

  • Requirements: Select product, insert money, dispense product and change. Handle: insufficient funds, out of stock, exact change only mode
  • States: Idle -> ProductSelected -> MoneyInserted -> Dispensing -> Idle
  • Patterns: State (machine lifecycle), Strategy (change calculation -- greedy denomination), Chain of Responsibility (validation: product exists? in stock? enough money?)

Focus area: Practice drawing the state machine diagram first (Phase 2), then implementing it cleanly. This should take less than 25 minutes of coding.

# Starter hint:
class VendingState(ABC):
    @abstractmethod
    def select_product(self, machine, product_id): pass
    @abstractmethod
    def insert_money(self, machine, amount): pass
    @abstractmethod
    def dispense(self, machine): pass
    @abstractmethod
    def cancel(self, machine): pass

Exercise 3: Mock Interview Self-Review

Pick any problem from this course (Parking Lot, Elevator, Snake Game, etc.) and solve it under interview conditions:

  • Set a 45-minute timer
  • Follow the 5-5-5-25-5 framework strictly
  • Talk out loud as you design (practice verbal communication)
  • When done, review against the 10-point checklist

Self-review checklist:

  • Did you spend at least 5 minutes on requirements before coding?
  • Did you identify all core entities (6-10)?
  • Did you use enums for status/type values (not raw strings)?
  • Did you justify each pattern choice with a reason and alternative?
  • Can you walk through a complete use case end-to-end?
  • Did you mention edge cases (even ones you did not implement)?
  • Did you finish within 45 minutes?

Tip: Record yourself (audio or video) and review. The biggest improvements come from hearing how you communicate your design decisions.
