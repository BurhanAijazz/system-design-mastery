UML to Code Translation

Translating between UML diagrams and Python code is a core LLD interview skill. This lesson walks through both directions: reading a UML diagram and writing code, and reading code and producing a diagram.

Translation Process

  1. Step 1: Create class stubs (empty classes)
  2. Step 2: Add attributes
  3. Step 3: Add methods
  4. Step 4: Implement relationships
  5. Step 5: Add business logic

Example: University System UML

+------------------+  1      1..*  +------------------+
|   University     | ◆-----------> |   Department     |
+------------------+               +------------------+
| - name: str      |               | - name: str      |
| - founded: int   |               | - budget: float  |
+------------------+               +------------------+
| + add_dept()     |               | + hire(prof)     |
| + get_dept()     |               | + get_staff()    |
+------------------+               +------------------+
                                        ◇
                                        | 0..*
                                        v
+------------------+  0..*   0..* +------------------+
|   Student        | <---------> |   Course         |
+------------------+             +------------------+
| - student_id: str|             | - code: str      |
| - name: str      |             | - name: str      |
| - gpa: float     |             | - credits: int   |
+------------------+             +------------------+
| + enroll(course) |             | + get_enrolled() |
+------------------+             +------------------+
                                        ^
                                        | teaches 1
                                  +------------------+
                                  |   Professor      |
                                  +------------------+
                                  | - name: str      |
                                  | - specialty: str |
                                  +------------------+
                                  | + assign_course()|
                                  +------------------+

Relationships:
  University ◆--> Department  (composition, 1 to 1..*)
  Department ◇--> Professor   (aggregation, 0..*)
  Student <---> Course        (association, many-to-many)
  Professor --> Course        (association, teaches)

Step-by-Step Translation

from dataclasses import dataclass, field
from typing import List, Optional

@dataclass
class Course:
    code: str
    name: str
    credits: int
    _enrolled_students: List["Student"] = field(default_factory=list, repr=False)
    _professor: Optional["Professor"] = field(default=None, repr=False)

    def get_enrolled(self) -> List["Student"]:
        return list(self._enrolled_students)

    def _add_student(self, student: "Student") -> None:
        if student not in self._enrolled_students:
            self._enrolled_students.append(student)


@dataclass
class Professor:
    name: str
    specialty: str
    _courses: List[Course] = field(default_factory=list, repr=False)

    def assign_course(self, course: Course) -> None:
        self._courses.append(course)
        course._professor = self


@dataclass
class Student:
    student_id: str
    name: str
    gpa: float = 0.0
    _courses: List[Course] = field(default_factory=list, repr=False)

    def enroll(self, course: Course) -> None:
        self._courses.append(course)
        course._add_student(self)


class Department:
    def __init__(self, name: str, budget: float):
        self.name = name
        self.budget = budget
        self._staff: List[Professor] = []

    def hire(self, prof: Professor) -> None:
        self._staff.append(prof)

    def get_staff(self) -> List[Professor]:
        return list(self._staff)


class University:
    def __init__(self, name: str, founded: int):
        self.name = name
        self.founded = founded
        self._departments: List[Department] = []  # composition

    def add_dept(self, name: str, budget: float) -> Department:
        dept = Department(name, budget)  # University creates departments
        self._departments.append(dept)
        return dept

    def get_dept(self, name: str) -> Optional[Department]:
        for dept in self._departments:
            if dept.name == name:
                return dept
        return None

Key Translation Rules

  • Composition (filled diamond): parent creates children in its constructor or factory methods
  • Aggregation (hollow diamond): parent receives children via method parameters
  • Many-to-many association: both sides hold lists of the other
  • Multiplicity 1..*: use a List that must not be empty
  • Multiplicity 0..1: use Optional type

Reverse: Code to UML

When reading code and producing a UML diagram:

  1. Identify classes (every class keyword)
  2. List attributes (look in __init__ or dataclass fields)
  3. List public methods (no underscore prefix)
  4. Identify relationships by examining attribute types (List, Optional) and method parameters
  5. Determine if relationships are dependency, association, aggregation, or composition
