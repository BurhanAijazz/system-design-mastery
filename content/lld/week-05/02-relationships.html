UML Relationships: Association, Aggregation, and Composition

Classes do not exist in isolation -- they have relationships. Understanding these relationships is critical for LLD interviews. There are six types, from weakest to strongest coupling.

1. Dependency (Uses Temporarily)

UML notation: dashed arrow (- - - ->). The weakest relationship -- just a method parameter or local variable.

class PaymentProcessor:
    def charge(self, amount: float) -> bool:
        print(f"Charged ${amount:.2f}")
        return True


class Order:
    def __init__(self, order_id: str, total: float):
        self.order_id = order_id
        self.total = total

    def checkout(self, processor: PaymentProcessor) -> bool:
        """PaymentProcessor is a parameter -- dependency, not stored."""
        return processor.charge(self.total)

The Order uses PaymentProcessor in a method but does not store it as a field.

2. Association (Has Reference To)

UML notation: solid arrow (----->). One class has a reference to another. Both can exist independently.

from dataclasses import dataclass, field
from typing import List

@dataclass
class Course:
    code: str
    name: str

@dataclass
class Student:
    name: str
    courses: List[Course] = field(default_factory=list)

    def enroll(self, course: Course) -> None:
        self.courses.append(course)

Deleting a Student does not delete the Courses. Both have independent lifecycles.

3. Aggregation (Weak Ownership)

UML notation: hollow diamond (◇----->). A "has-a" relationship where the contained object can exist without the container.

@dataclass
class Player:
    name: str
    position: str

class Team:
    def __init__(self, name: str):
        self.name = name
        self._players: List[Player] = []

    def add_player(self, player: Player) -> None:
        self._players.append(player)

    def remove_player(self, player: Player) -> None:
        self._players.remove(player)

A Player can exist without a Team (free agent). A Team has players but does not own their lifecycle.

4. Composition (Strong Ownership)

UML notation: filled diamond (◆----->). The strongest "has-a" relationship -- the contained object's lifecycle is bound to the container. When the container is destroyed, so are its parts.

class Room:
    def __init__(self, name: str, area: float):
        self.name = name
        self.area = area

class House:
    def __init__(self, address: str):
        self.address = address
        # Rooms are CREATED inside House -- composition
        self._rooms: List[Room] = [
            Room("Living Room", 30.0),
            Room("Kitchen", 15.0),
            Room("Bedroom", 20.0),
        ]

Rooms are created inside House and destroyed when House is destroyed. A Room cannot exist without its House.

5. Inheritance (Is-a)

UML notation: hollow triangle arrow (-----▷). A subclass extends the behavior of a parent class.

from abc import ABC, abstractmethod

class Vehicle(ABC):
    def __init__(self, make: str, year: int):
        self.make = make
        self.year = year

    @abstractmethod
    def fuel_type(self) -> str:
        pass

class Car(Vehicle):
    def fuel_type(self) -> str:
        return "gasoline"

class ElectricCar(Vehicle):
    def fuel_type(self) -> str:
        return "electric"

6. Realization (Implements Interface)

UML notation: dashed triangle arrow. A class implements an abstract interface.

class Sortable(ABC):
    @abstractmethod
    def sort_key(self) -> str:
        pass

class Employee(Sortable):
    def __init__(self, name: str):
        self.name = name

    def sort_key(self) -> str:
        return self.name

Multiplicity

UML shows how many objects participate in a relationship:

  • 1 -- exactly one
  • 0..1 -- zero or one (Optional)
  • 0..* or * -- zero or more (List)
  • 1..* -- one or more (non-empty List)

Quick Reference: Choosing the Right Relationship

  • Object passed as parameter? Dependency
  • Object stored as field, independent lifecycle? Association
  • Object stored, can exist without parent? Aggregation
  • Object created by parent, dies with parent? Composition
  • Object is a specialized version? Inheritance
  • Object fulfills a contract? Realization
