Design: Online Stock Span

Requirements

  • Process a stream of daily stock prices one at a time
  • Return the span: number of consecutive days (including today) the price was ≤ today's price
  • O(1) amortized per call
  • Encapsulate logic in a clean OOP class with stack internals

Approach

  • Maintain a monotonic decreasing stack of (price, span) pairs
  • On each new price, pop all stack entries with price ≤ current, accumulating their spans
  • Push the new price with its total accumulated span
  • Each element is pushed and popped at most once: amortized O(1)

Key Code

class StockSpanner:
    def __init__(self):
        self._stack: list[tuple[int, int]] = []  # (price, span)

    def next(self, price: int) -> int:
        span = 1
        while self._stack and self._stack[-1][0] <= price:
            span += self._stack.pop()[1]
        self._stack.append((price, span))
        return span


# Usage
spanner = StockSpanner()
prices = [100, 80, 60, 70, 60, 75, 85]
spans  = [spanner.next(p) for p in prices]
# spans: [1, 1, 1, 2, 1, 4, 6]

Takeaways

  • Monotonic stacks solve "nearest greater/smaller" problems in amortized O(1)
  • Storing accumulated spans avoids re-scanning previous days
  • Pattern applies to temperature problems, histogram areas, and sliding-window max
  • Wrapping in a class provides a clean streaming API for real-time data
