Design: Twitter Feed

Requirements

  • Post a tweet, follow/unfollow users
  • Retrieve the 10 most recent tweets in a user's feed
  • Feed includes own tweets plus tweets from followed users
  • Apply Observer for follow graph, Strategy for feed ranking, merge K sorted lists

Approach

  • Each user maintains a list of tweets sorted by timestamp (descending)
  • Follow relationships stored as adjacency sets (Observer-like subscription)
  • Feed generation: merge K sorted tweet lists using a min-heap
  • Strategy pattern allows swapping chronological vs. engagement-based ranking

Key Code

import heapq
from collections import defaultdict

class Twitter:
    def __init__(self):
        self._ts = 0
        self._tweets = defaultdict(list)   # userId -> [(ts, tweetId)]
        self._follows = defaultdict(set)   # userId -> {followeeIds}

    def post_tweet(self, user_id: int, tweet_id: int):
        self._ts += 1
        self._tweets[user_id].append((-self._ts, tweet_id))

    def get_news_feed(self, user_id: int) -> list[int]:
        sources = self._follows[user_id] | {user_id}
        heap = []
        for uid in sources:
            tw = self._tweets[uid]
            if tw:
                idx = len(tw) - 1
                heapq.heappush(heap, (tw[idx][0], tw[idx][1], uid, idx))
        result = []
        while heap and len(result) < 10:
            ts, tid, uid, idx = heapq.heappop(heap)
            result.append(tid)
            if idx > 0:
                nxt = self._tweets[uid][idx - 1]
                heapq.heappush(heap, (nxt[0], nxt[1], uid, idx - 1))
        return result

    def follow(self, follower: int, followee: int):
        self._follows[follower].add(followee)

    def unfollow(self, follower: int, followee: int):
        self._follows[follower].discard(followee)

Takeaways

  • Merge K sorted lists is the core algorithm for fan-out-on-read feeds
  • Observer pattern maps naturally to follow/unfollow subscriptions
  • Strategy pattern enables A/B testing different ranking algorithms
  • At scale, fan-out-on-write (pre-computed feeds) reduces read latency
