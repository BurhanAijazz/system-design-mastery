Design: Calendar Booking System

Requirements

  • Book events with start/end times; detect conflicts
  • Support recurring events (daily, weekly, monthly)
  • Handle timezone conversions for multi-region users
  • Efficient overlap queries via interval tree

Approach

  • Store events in a balanced BST (interval tree) keyed by start time
  • Conflict detection: query all intervals overlapping [new_start, new_end)
  • Recurring events expanded on-read within a query window (virtual instances)
  • All times stored in UTC; convert to user timezone at the API layer

Key Code

from datetime import datetime, timedelta
from sortedcontainers import SortedList

class Event:
    def __init__(self, start: datetime, end: datetime, title: str):
        self.start = start
        self.end = end
        self.title = title

class Calendar:
    def __init__(self):
        self._events = SortedList(key=lambda e: e.start)

    def has_conflict(self, start: datetime, end: datetime) -> bool:
        for ev in self._events:
            if ev.start >= end:
                break
            if ev.end > start:
                return True
        return False

    def book(self, start: datetime, end: datetime, title: str) -> Event:
        if self.has_conflict(start, end):
            raise ValueError("Conflicting event exists")
        ev = Event(start, end, title)
        self._events.add(ev)
        return ev

    def events_in_range(self, start: datetime, end: datetime) -> list[Event]:
        return [e for e in self._events if e.end > start and e.start < end]

class RecurringEvent:
    def __init__(self, base: Event, freq: timedelta, until: datetime):
        self.base = base
        self.freq = freq
        self.until = until

    def expand(self, window_start: datetime, window_end: datetime):
        instances, current = [], self.base.start
        duration = self.base.end - self.base.start
        while current <= self.until and current < window_end:
            inst_end = current + duration
            if inst_end > window_start:
                instances.append(Event(current, inst_end, self.base.title))
            current += self.freq
        return instances

Takeaways

  • Sorted containers give O(log n) insertion and efficient range queries
  • Expanding recurring events lazily avoids storing infinite instances
  • UTC storage + display-layer conversion prevents timezone bugs
  • Double-booking variants (allow K overlaps) reuse the same conflict-check pattern
