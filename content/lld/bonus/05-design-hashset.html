Design: HashSet

Requirements

  • Support add(key), remove(key), contains(key)
  • No duplicate elements allowed
  • Amortized O(1) for all operations
  • Demonstrate composition over inheritance

Approach

  • Internally delegate to a HashMap where values are a sentinel
  • Composition: HashSet has-a HashMap, not is-a HashMap
  • Same bucket-array + chaining strategy under the hood
  • Alternative: use a simple boolean array for constrained key ranges

Key Code

class MyHashSet:
    """HashSet built by composing a HashMap."""
    _PRESENT = object()  # sentinel value

    def __init__(self):
        self._map = MyHashMap()

    def add(self, key: int) -> None:
        self._map.put(key, self._PRESENT)

    def remove(self, key: int) -> None:
        self._map.remove(key)

    def contains(self, key: int) -> bool:
        return self._map.get(key) is self._PRESENT


# --- Lightweight alternative for key range [0, 10^6] ---
class BoolHashSet:
    def __init__(self):
        self._data = [False] * 1_000_001

    def add(self, key):  self._data[key] = True
    def remove(self, key): self._data[key] = False
    def contains(self, key): return self._data[key]

Takeaways

  • Composition keeps the public interface minimal (no value semantics leak)
  • Sentinel object avoids confusing "not found" with a stored value
  • Boolean-array approach works when the key space is small and dense
  • Same design relationship exists in Java: HashSet wraps HashMap
