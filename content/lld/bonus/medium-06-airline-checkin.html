Design: Airline Check-in

Requirements

  • Passengers transition through states: NotCheckedIn -> CheckedIn -> Boarded
  • Seat assignment via a configurable Strategy (window-first, random, zone-based)
  • Boarding pass generation upon successful check-in
  • Prevent invalid transitions (e.g., board before check-in)

Approach

  • State pattern: each state class defines allowed transitions and actions
  • Strategy pattern: seat assignment algorithm is injected at runtime
  • Flight maintains available seats; check-in assigns and removes from pool
  • State objects enforce business rules without if/else chains

Key Code

from abc import ABC, abstractmethod

class SeatStrategy(ABC):
    @abstractmethod
    def assign(self, available: list[str]) -> str: ...

class WindowFirstStrategy(SeatStrategy):
    def assign(self, available):
        for s in available:
            if s.endswith(("A", "F")):
                return s
        return available[0]

class PassengerState(ABC):
    @abstractmethod
    def check_in(self, ctx): ...
    @abstractmethod
    def board(self, ctx): ...

class NotCheckedIn(PassengerState):
    def check_in(self, ctx):
        seat = ctx.strategy.assign(ctx.flight.available_seats)
        ctx.flight.available_seats.remove(seat)
        ctx.seat = seat
        ctx.state = CheckedIn()
    def board(self, ctx):
        raise RuntimeError("Must check in first")

class CheckedIn(PassengerState):
    def check_in(self, ctx):
        raise RuntimeError("Already checked in")
    def board(self, ctx):
        ctx.state = Boarded()

class Boarded(PassengerState):
    def check_in(self, ctx): raise RuntimeError("Already boarded")
    def board(self, ctx): raise RuntimeError("Already boarded")

class Passenger:
    def __init__(self, name, flight, strategy: SeatStrategy):
        self.name = name
        self.flight = flight
        self.strategy = strategy
        self.seat = None
        self.state: PassengerState = NotCheckedIn()

    def check_in(self): self.state.check_in(self)
    def board(self):    self.state.board(self)

Takeaways

  • State pattern eliminates fragile status-flag conditionals
  • Strategy pattern makes seat-assignment rules hot-swappable
  • Each state class is small, testable, and follows single-responsibility
  • Extend with Observer to notify gate agents on boarding events
