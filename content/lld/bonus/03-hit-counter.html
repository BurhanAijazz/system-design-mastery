Design: Hit Counter

Requirements

  • Record a hit at a given timestamp
  • Return total hits in the last 300 seconds (5 minutes)
  • Timestamps arrive in non-decreasing order
  • Support high-throughput concurrent hits

Approach

  • Sliding window via a circular buffer of size 300
  • Each slot stores a (timestamp, count) pair
  • On hit(t): map t % 300 to a slot; reset if stale, else increment
  • On getHits(t): sum all slots whose timestamp is within [t-299, t]
  • O(1) hit, O(300)=O(1) getHits, O(300) space

Key Code

class HitCounter:
    def __init__(self):
        self._times = [0] * 300
        self._hits  = [0] * 300

    def hit(self, timestamp: int) -> None:
        idx = timestamp % 300
        if self._times[idx] != timestamp:
            self._times[idx] = timestamp
            self._hits[idx] = 1
        else:
            self._hits[idx] += 1

    def get_hits(self, timestamp: int) -> int:
        total = 0
        for i in range(300):
            if timestamp - self._times[i] < 300:
                total += self._hits[i]
        return total

Takeaways

  • Circular buffers give constant-space sliding windows
  • Modular indexing eliminates the need to shift data
  • For thread safety, each slot can use an independent lock
  • Pattern extends to any fixed-window counting problem
