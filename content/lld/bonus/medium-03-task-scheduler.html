Design: Task Scheduler

Requirements

  • Schedule tasks with a cooldown interval n between same-type tasks
  • Return the minimum number of time slots to finish all tasks
  • Use a priority queue to always pick the most frequent task
  • Apply Strategy pattern for scheduling policies; Observer for completion events

Approach

  • Count task frequencies; push into a max-heap
  • Each round processes up to n+1 distinct tasks
  • Tasks that still have remaining count go into a cooldown queue
  • Idle slots fill when fewer than n+1 tasks are available

Key Code

import heapq
from collections import Counter

class TaskScheduler:
    def __init__(self, cooldown: int):
        self._cooldown = cooldown
        self._observers = []

    def subscribe(self, observer):
        self._observers.append(observer)

    def _notify(self, msg):
        for obs in self._observers:
            obs.on_event(msg)

    def least_interval(self, tasks: list[str]) -> int:
        heap = [-c for c in Counter(tasks).values()]
        heapq.heapify(heap)
        time = 0
        while heap:
            cycle, temp = self._cooldown + 1, []
            for _ in range(cycle):
                if heap:
                    cnt = heapq.heappop(heap) + 1  # +1 because stored negative
                    if cnt:
                        temp.append(cnt)
                time += 1
                if not heap and not temp:
                    break
            for t in temp:
                heapq.heappush(heap, t)
        self._notify("all_tasks_complete")
        return time

Takeaways

  • Greedy + max-heap ensures most-frequent tasks are spaced optimally
  • Strategy pattern lets you swap scheduling policies (round-robin, priority, deadline)
  • Observer decouples completion notification from scheduling logic
  • Formula shortcut: (max_freq - 1) * (n + 1) + count_of_max_freq_tasks
