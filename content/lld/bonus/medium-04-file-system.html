Design: In-Memory File System

Requirements

  • Support mkdir, ls, add_content_to_file, read_content_from_file
  • Paths are Unix-style: /a/b/c
  • Apply Composite pattern: directories contain files or other directories
  • ls on a file returns its name; on a directory returns sorted children

Approach

  • A Node base with Directory and File subclasses (Composite)
  • Traverse the trie of directories by splitting the path on /
  • mkdir -p semantics: create intermediate directories automatically
  • Files store appended string content

Key Code

from collections import defaultdict

class Node:
    pass

class File(Node):
    def __init__(self):
        self.content = ""

class Directory(Node):
    def __init__(self):
        self.children: dict[str, Node] = {}

class FileSystem:
    def __init__(self):
        self.root = Directory()

    def _traverse(self, path: str) -> Node:
        node = self.root
        for part in path.split("/")[1:]:
            if part:
                node = node.children[part]
        return node

    def mkdir(self, path: str):
        node = self.root
        for part in path.split("/")[1:]:
            if part and part not in node.children:
                node.children[part] = Directory()
            node = node.children[part] if part else node

    def ls(self, path: str) -> list[str]:
        node = self._traverse(path)
        if isinstance(node, File):
            return [path.split("/")[-1]]
        return sorted(node.children.keys())

    def add_content(self, path: str, content: str):
        parts = path.split("/")
        self.mkdir("/".join(parts[:-1]))
        parent = self._traverse("/".join(parts[:-1]))
        name = parts[-1]
        if name not in parent.children:
            parent.children[name] = File()
        parent.children[name].content += content

    def read_content(self, path: str) -> str:
        return self._traverse(path).content

Takeaways

  • Composite pattern lets files and directories share a uniform interface
  • Trie-like directory structure mirrors real filesystem internals
  • Append semantics on files simplify concurrent-write reasoning
  • Extend with permissions, symlinks, or Observer for watch notifications
