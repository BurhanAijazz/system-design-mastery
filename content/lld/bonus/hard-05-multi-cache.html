Design: Multi-Level Cache

Requirements

  • L1 (small/fast), L2 (medium), L3 (large/slow) cache hierarchy
  • Lookup traverses levels via Chain of Responsibility
  • Write-through: writes propagate down to all levels
  • Each level can have its own eviction policy (LRU, LFU, etc.)

Approach

  • Each cache level implements a common interface with get/put
  • Chain of Responsibility: on miss, delegate to the next level; on hit, backfill upper levels
  • Write-through: put writes to current level and forwards to successor
  • Each level wraps an LRU (or other) cache with its own capacity

Key Code

from collections import OrderedDict
from abc import ABC, abstractmethod

class CacheLevel(ABC):
    @abstractmethod
    def get(self, key): ...
    @abstractmethod
    def put(self, key, value): ...

class LRUCacheLevel(CacheLevel):
    def __init__(self, capacity: int, name: str, next_level: CacheLevel = None):
        self.cap = capacity
        self.name = name
        self.next = next_level
        self._store = OrderedDict()

    def get(self, key):
        if key in self._store:
            self._store.move_to_end(key)
            return self._store[key]
        if self.next:
            val = self.next.get(key)
            if val is not None:
                self.put_local(key, val)  # backfill
                return val
        return None

    def put_local(self, key, value):
        if key in self._store:
            self._store.move_to_end(key)
        self._store[key] = value
        if len(self._store) > self.cap:
            self._store.popitem(last=False)

    def put(self, key, value):
        self.put_local(key, value)
        if self.next:  # write-through
            self.next.put(key, value)


# Build L3 -> L2 -> L1 chain
l3 = LRUCacheLevel(capacity=1000, name="L3")
l2 = LRUCacheLevel(capacity=100,  name="L2", next_level=l3)
l1 = LRUCacheLevel(capacity=10,   name="L1", next_level=l2)

l1.put("x", 42)       # writes through L1 -> L2 -> L3
assert l1.get("x") == 42

Takeaways

  • Chain of Responsibility cleanly models hierarchical cache lookup
  • Backfill on miss keeps hot data in faster levels automatically
  • Write-through ensures consistency; write-back trades it for lower write latency
  • Each level's eviction policy is independent and swappable via Strategy
