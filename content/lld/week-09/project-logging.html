Project: Logging Framework

A full LLD implementation of a logging framework similar to Python's built-in logging module. This project combines six design patterns: Singleton for the logger registry, Strategy for formatters, Chain of Responsibility for filters, Observer for handlers, Builder for configuration, and Composite for handler groups.

Requirements

Functional:

  • Log messages at different levels (DEBUG, INFO, WARNING, ERROR, CRITICAL)
  • Route messages to multiple handlers (console, file, memory)
  • Format messages differently per handler (simple, JSON, detailed)
  • Filter messages via chain (level filter, keyword filter, rate limit filter)
  • Named loggers with hierarchy (app -> app.db propagation)
  • Global logger registry (singleton)
  • Builder pattern for easy configuration
  • Thread-safe logging

Out of scope: Actual file I/O (simulated), log rotation, remote logging.

Core Entities

class LogLevel(IntEnum):
    DEBUG = 10
    INFO = 20
    WARNING = 30
    ERROR = 40
    CRITICAL = 50

@dataclass
class LogRecord:
    level: LogLevel
    message: str
    logger_name: str
    timestamp: datetime = field(default_factory=datetime.now)
    extra: Dict = field(default_factory=dict)

Design Patterns Used

  • Singleton -- LoggerRegistry ensures a single global registry. Calling get_logger("app") always returns the same Logger instance
  • Strategy -- Formatters (SimpleFormatter, JSONFormatter, DetailedFormatter) convert LogRecords to strings. Each handler can use a different formatter
  • Chain of Responsibility -- Filters form a chain: LevelFilter -> KeywordFilter -> RateFilter. A record must pass all filters to reach handlers
  • Observer -- Handlers (ConsoleHandler, FileHandler, MemoryHandler) subscribe to a logger. When a record passes filters, all handlers are notified
  • Builder -- LoggerBuilder provides a fluent API: builder.name("app").level(INFO).add_handler(console).build()
  • Composite -- Handler groups let you treat a collection of handlers as a single handler

Key Interfaces

class Formatter(ABC):
    @abstractmethod
    def format(self, record: LogRecord) -> str:
        pass

class Handler(ABC):
    @abstractmethod
    def emit(self, record: LogRecord) -> None:
        pass

class Filter(ABC):
    @abstractmethod
    def should_log(self, record: LogRecord) -> bool:
        pass

Implementation Steps

  • Define LogLevel (IntEnum) and LogRecord (dataclass)
  • Implement three Formatters: Simple, JSON, Detailed
  • Implement three Handlers: Console, File, Memory
  • Implement three Filters: Level, Keyword, Rate
  • Build the Logger class that chains filters and dispatches to handlers
  • Create the LoggerRegistry singleton with get_logger()
  • Add the Builder for fluent configuration
  • Add thread safety with threading.Lock in handlers
