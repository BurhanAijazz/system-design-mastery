Cache Design: LRU, LFU, and TTL

Caching is one of the most commonly asked LLD topics. This lesson covers the core cache data structures, eviction strategies (LRU, LFU, TTL), and the Strategy pattern for swapping eviction policies. The key goal: O(1) get and put operations with bounded memory and automatic eviction.

Cache Interface

Every cache implementation needs these core operations:

  • get(key) -- return the value or None
  • put(key, value) -- insert or update an entry
  • size() -- current number of entries
  • Automatic eviction when the cache exceeds its capacity
class Cache(ABC):
    @abstractmethod
    def get(self, key: str) -> Optional[Any]:
        pass

    @abstractmethod
    def put(self, key: str, value: Any) -> None:
        pass

    @abstractmethod
    def size(self) -> int:
        pass

LRU Cache (Doubly-Linked List + HashMap)

Least Recently Used: evict the entry that has not been accessed for the longest time. This is the number one most asked cache question in interviews.

Data structure: Doubly-linked list + HashMap

  • HashMap maps key to DLL Node for O(1) lookup
  • Doubly-linked list maintains access order (most recent at head, least at tail)
  • On get: move the accessed node to the head
  • On put: add to head, evict from tail if full
  • Sentinel head/tail nodes eliminate null checks
class DLLNode:
    """Doubly-linked list node."""
    def __init__(self, key: str = "", value: Any = None):
        self.key = key
        self.value = value
        self.prev: Optional[DLLNode] = None
        self.next: Optional[DLLNode] = None


class LRUCache(Cache):
    """LRU Cache with O(1) get/put using DLL + HashMap."""

    def __init__(self, capacity: int):
        self._capacity = capacity
        self._map: Dict[str, DLLNode] = {}
        # Sentinel nodes (avoid null checks)
        self._head = DLLNode()  # Most recently used
        self._tail = DLLNode()  # Least recently used
        self._head.next = self._tail
        self._tail.prev = self._head

LFU Cache

Least Frequently Used: evict the entry with the lowest access count. When there is a tie in frequency, evict the least recently used among them. Uses frequency buckets where each frequency maps to an OrderedDict for LRU tie-breaking.

TTL Cache

Time-To-Live: entries expire after a configured duration. Two expiration strategies:

  • Lazy expiration -- check on access and remove if expired (simple, but stale entries linger)
  • Active expiration -- background thread periodically scans and removes expired entries

Strategy Pattern for Eviction

Use the Strategy pattern to make the eviction policy swappable. The cache delegates eviction decisions to an EvictionStrategy, making it easy to switch between LRU, LFU, or TTL without changing the cache shell.

Key Design Takeaways

  • LRU is the default interview answer -- know the DLL + HashMap implementation cold
  • Both get and put must be O(1) -- this drives the choice of data structures
  • Sentinel nodes in the DLL eliminate edge-case null checks
  • LFU adds frequency tracking; use OrderedDict per frequency for O(1) LRU tie-breaking
  • TTL caches trade freshness for complexity; lazy expiration is simpler, active is cleaner
  • Track cache statistics (hits, misses, evictions) to evaluate effectiveness
