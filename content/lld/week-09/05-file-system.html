In-Memory File System Design

A classic LLD problem that uses the Composite pattern to model the tree structure of files and directories. This lesson covers file system modeling, path resolution, permissions, directory traversal, and search.

Permissions Model

Simplified Unix-style permissions using Flag enum. Permissions are composable: READ | WRITE gives read and write access.

class Permission(Flag):
    NONE = 0
    READ = auto()
    WRITE = auto()
    EXECUTE = auto()
    ALL = READ | WRITE | EXECUTE

# Composable:
# READ | WRITE -> read+write
# READ in ALL -> True

FileSystemNode (Composite Pattern)

The Composite pattern models the tree structure where both files and directories share a common interface:

  • FileSystemNode (abstract) -- common interface with name, path, permissions, owner, timestamps
  • File (leaf) -- has content and reports its own size
  • Directory (composite) -- has children, can contain files and other directories, size is sum of children
class FileSystemNode(ABC):
    """Abstract node in the file system tree."""

    def __init__(self, name: str, owner: str = "root",
                 permissions: Permission = Permission.ALL):
        self.name = name
        self.owner = owner
        self.permissions = permissions
        self.created_at = datetime.now()
        self.modified_at = datetime.now()
        self._parent: Optional[Directory] = None

    @property
    def path(self) -> str:
        if self._parent is None:
            return "/" if self.name == "/" else f"/{self.name}"
        parent_path = self._parent.path
        if parent_path == "/":
            return f"/{self.name}"
        return f"{parent_path}/{self.name}"

    @abstractmethod
    def get_size(self) -> int:
        pass

File and Directory

class File(FileSystemNode):
    def __init__(self, name, content="", **kwargs):
        super().__init__(name, **kwargs)
        self.content = content

    def get_size(self) -> int:
        return len(self.content)


class Directory(FileSystemNode):
    def __init__(self, name, **kwargs):
        super().__init__(name, **kwargs)
        self._children: Dict[str, FileSystemNode] = {}

    def add(self, node: FileSystemNode) -> None:
        self._children[node.name] = node
        node._parent = self

    def get_size(self) -> int:
        return sum(child.get_size() for child in self._children.values())

Path Resolution

The FileSystem class provides a resolve(path) method that walks the tree by splitting the path on "/" and following each component through the directory children. Handle edge cases: root path "/", trailing slashes, non-existent paths.

Directory Traversal

Implement traversal using both DFS and BFS with the Iterator pattern:

  • DFS (depth-first) -- useful for finding files deep in the tree, uses a stack
  • BFS (breadth-first) -- useful for finding files at a certain depth level, uses a queue

Key Design Takeaways

  • Composite pattern is the natural fit for tree structures like file systems
  • Flag enum makes permissions composable and easy to check with the in operator
  • Path property is computed by walking up the parent chain -- no need to store full paths
  • get_size() demonstrates the power of Composite: directory delegates to children recursively
  • Permission checks should verify the user has the required flag before any operation
