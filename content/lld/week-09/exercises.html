These exercises reinforce the infrastructure system patterns covered this week: cache design, rate limiting, logging frameworks, pub/sub messaging, and file system modeling. Focus on choosing the right data structures and composing patterns effectively.

Exercise 1: LRU Cache with TTL

Implement a cache that combines LRU eviction with time-to-live expiration. Requirements:

  • O(1) get and put operations using a doubly-linked list and HashMap
  • Each entry has a configurable TTL (time-to-live in seconds)
  • On get(key): if the entry exists but is expired, remove it and return None
  • On put(key, value, ttl): insert with the specified TTL, evict LRU if at capacity
  • Track and report cache statistics: hits, misses, evictions, expirations

Key patterns: DLL + HashMap for O(1) operations, lazy expiration for TTL, dataclass for CacheStats.

# Starter hint:
class DLLNode:
    def __init__(self, key="", value=None, expires_at=None):
        self.key = key
        self.value = value
        self.expires_at = expires_at  # timestamp or None
        self.prev = None
        self.next = None

# Implement LRUWithTTLCache(Cache):
#   - get(): check expiry, move to head if valid
#   - put(): insert at head, evict from tail if full
#   - _is_expired(node): check against current time

Exercise 2: Composed Rate Limiter

Build a rate limiting system that supports layered rules. Requirements:

  • Support at least two algorithms: Token Bucket and Fixed Window Counter
  • Allow composing multiple rules: e.g., 10 requests/second AND 1000 requests/hour
  • A request is allowed only if ALL rules pass (Chain of Responsibility)
  • Return a RateLimitResult with allowed status, remaining count, and retry_after
  • Create a @rate_limited decorator that applies rate limiting to any function

Key patterns: Strategy for algorithm selection, Chain of Responsibility for composing rules, Decorator for transparent application.

# Starter hint:
@dataclass
class RateLimitRule:
    name: str
    max_requests: int
    window_seconds: float
    algorithm: str  # "token_bucket" or "fixed_window"

class ComposedLimiter:
    def __init__(self, rules: List[RateLimitRule]):
        self._limiters = [self._create(r) for r in rules]

    def allow_request(self, client_id: str) -> RateLimitResult:
        # ALL limiters must allow

Exercise 3: Mini Pub/Sub System

Implement a simple in-memory pub/sub message broker with these features:

  • Topics with subscribe and unsubscribe operations
  • Publish a message to a topic and fan-out to all subscribers
  • At-least-once delivery: retry up to 3 times on subscriber failure
  • Dead letter queue for messages that exhaust all retries
  • Message priority: CRITICAL messages are delivered before NORMAL ones

Key patterns: Observer for fan-out, Priority enum for ordering, DLQ for failure handling.

class MessageBroker:
    def __init__(self):
        self._topics: Dict[str, List[Subscriber]] = {}
        self._dlq: List[Message] = []
