Exercise 1: Hotel Room Booking System

Design a hotel room booking system that supports the full reservation lifecycle. Guests should be able to search for available rooms by date range, make a reservation (with an expiration timer), confirm by payment, check in, and check out.

  • Model rooms with types: Single, Double, Suite
  • Implement the reservation lifecycle: PENDING, CONFIRMED, ACTIVE (checked in), COMPLETED (checked out), CANCELLED, EXPIRED
  • Use a pricing strategy for room rates (flat per night, weekend surcharge, seasonal pricing)
  • Pending reservations expire after 15 minutes if not confirmed
  • Prevent double-booking: a room cannot be reserved by two guests for overlapping dates

Exercise 2: Movie Ticket Booking with Payments

Design a movie ticket booking system that combines seat selection with payment processing. Users browse showtimes, select seats, pay, and receive a confirmation.

  • Model a theater with screens, each screen has rows and seats
  • Showtimes link a movie to a screen at a specific date and time
  • Support multiple payment methods (CreditCard, Wallet, UPI) using the Strategy pattern
  • Seats are held for 5 minutes during payment processing -- release if payment fails or times out
  • Implement tiered pricing: front rows cheaper, middle premium, back standard

Exercise 3: Warehouse Inventory Tracker

Design an inventory management system for a warehouse with multiple zones and bin locations. Items arrive, get stored in bins, and are picked for outbound orders.

  • Warehouse has zones (e.g., "A", "B", "C"), each zone has bins (slots)
  • Bins have a type (small, medium, large) and a capacity
  • Use an allocation strategy to place incoming items: first-available, nearest-to-dock, or best-fit by size
  • Track stock levels per product: total quantity, reserved quantity, available quantity
  • Support restocking (inbound) and picking (outbound) operations
  • Bins can be put into maintenance mode (cleaning, repair)

Tips

  • Start by identifying entities and their relationships (Step 2 of the LLD framework)
  • Map state transitions for lifecycle objects (reservations, seats, bins)
  • Apply Strategy for anything that has multiple algorithms (pricing, allocation, payment)
  • Apply Observer for notifications (booking confirmed, stock low, payment processed)
  • Consider concurrency: what happens if two users try to book the same seat or bin?
